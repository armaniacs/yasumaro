## なぜ「信頼度」を表示しようと思ったか

Yasumaro は、Web ページを自動で記録する拡張機能です。自動化があると、ユーザーは次第に「今開いているページが本当に記録されていいのか」意識しなくなります。

私自身も、ある金融機関のサイトを開いているときに気づきました。

**「これ、勝手に記録されてたらまずくないか？」**

すべてを記録するのではなく、記録する前に「少し立ち止まるきっかけ」を作りたい。そこで Yasumaro に **ドメイン信頼度判定** 機能を追加しました。

この記事では、その設計思想から実装の仕組み、実際の使い方までを整理します。

## 結論から言うと

- Yasumaro は、開いているサイトのドメインを **4 段階** で評価している
- 評価結果は **ポップアップに表示** され、金融サイトや未検証サイトへの注意喚起になる
- **記録を自動でブロックするわけではない**（`LOCKED` レベル以外は警告表示が中心）
- 判定に使うデータは拡張機能内にあり、外部には送られない

つまり、「Trust」表示はあくまで **記録する前の目安** です。最終的な判断はユーザーが行います。

## 4 つの Trust Level

ドメインは以下のレベルに分類されます。

| レベル | 色 | どんなドメインか |
|---|---|---|
| **TRUSTED** | 緑 | 政府機関、大学、Tranco 上位ドメインなど |
| **SENSITIVE** | 黄 | 金融、ゲーム、SNS、ユーザーが追加した警戒ドメイン |
| **UNVERIFIED** | 灰 | いずれの信頼リストにも含まれないドメイン |
| **LOCKED** | グレー（暗め） | ブロック対象ドメイン。現時点では通常の判定フローでは返されず、将来の機能拡張用に予約されています |

`TRUSTED` は安心して記録できる目安、`SENSITIVE` は「内容を確認してから」、`UNVERIFIED` は「慎重に」というニュアンスです。

`LOCKED` は特別なレベルで、TrustChecker がこのレベルを受け取ると記録をブロックします。ただし、現時点の通常判定フローでは `LOCKED` は返されません。スキーマ上は将来の機能拡張用に用意されている状態です。

## 判定は 3 段階で行われる

Yasumaro のドメイン信頼度判定は、**3-Step Verification** と呼ばれる 3 段階のチェックで成り立っています。最初に一致した段階で結果が確定します。

```
入力された URL
    │
    ▼
Step 1: JP-Anchor TLD 判定
    │ 一致 → TRUSTED
    │
    ▼
Step 2: Sensitive List 判定
    │ 一致 → SENSITIVE
    │
    ▼
Step 3: Tranco 判定
    │ 一致 → TRUSTED
    │
    ▼
UNVERIFIED
```

### Step 1: JP-Anchor TLD

最初に、ドメインの末尾が日本の公式・公共 TLD と一致するかを見ます。

初期設定では以下が対象です。

- `.go.jp`（政府機関）
- `.ac.jp`（教育機関）
- `.lg.jp`（地方公共団体）

これらは改ざんリスクが低く、公式情報源として信頼できるため、最初に `TRUSTED` と判定されます。ユーザーが独自の TLD（例: `.ed.jp`）を追加することもできます。

### Step 2: Sensitive List

次に、金融・ゲーム・SNS に該当するかを確認します。判定順序には意味があります。

1. **Whitelist** に入っていれば `TRUSTED`
2. **ユーザーブラックリスト** に入っていれば `SENSITIVE`
3. **プリセット**（金融 / ゲーム / SNS）に一致すれば `SENSITIVE`

Whitelist が最初にチェックされるのは、たとえプリセットに含まれていても「このドメインは私は信頼している」というユーザーの意志を優先するためです。

### Step 3: Tranco

最後に、[Tranco List](https://tranco-list.eu/) という Web サイトの人気ランキングと照合します。Tranco は研究コミュニティで広く使われているリストで、上位に入るドメインは比較的信頼できるとされています。

判定時にはサブドメインも親ドメインに展開されます。たとえば `edition.cnn.com` であれば、`cnn.com` も候補として評価されます。

ただし、ここで重要なポイントがあります。

> **Tranco 判定を有効にするには、ダッシュボードの「Trust」タブから「Tranco List を更新」を実行する必要があります。**

初期状態では Tranco リストが空なので、更新しない限り Step 3 はスキップされ、JP-Anchor と Sensitive List の判定のみが行われます。

更新には `tranco-list.eu` へのアクセスが必要で、初回は同意を求められます。同意しない場合は 30 日後に再確認が表示されますが、その間も JP-Anchor と Sensitive List は動作し続けます。

## Safety Mode で信頼の幅を調整する

Tranco のどこまでを信頼するかは、**Safety Mode** で調整できます。

| Safety Mode | Tranco Tier | 信頼する範囲 |
|---|---|---|
| Strict | Top 1,000 | 上位 1,000 ドメインのみ |
| Balanced | Top 10,000 | 上位 10,000 ドメインまで（初期設定） |
| Relaxed | Top 100,000 | 上位 100,000 ドメインまで |

Strict にすれば厳選されたドメインだけが `TRUSTED` になり、Relaxed にすればより多くのドメインが `TRUSTED` になります。Safety Mode を変えると Tranco Tier も連動しますが、Tranco Tier は個別に変更することもできます。

私自身は普段 Balanced を使っています。信頼範囲が狭すぎると、見慣れたサイトでも UNVERIFIED になってしまい、逆に注意が麻痺する気がするからです。

## 警告設定を自分好みにする

ポップアップに警告を出す条件もカスタマイズできます。

| 設定 | 初期値 | 動作 |
|---|---|---|
| 金融サイトの警告 | ON | `finance` カテゴリで警告 |
| 警戒ドメインの警告 | ON | `gaming` / `sns` カテゴリで警告 |
| 未検証サイトの警告 | OFF | `UNVERIFIED` で警告 |

初期設定では、金融サイトとゲーム・SNS サイトで警告が表示されます。未検証サイトの警告は OFF なので、普段のブラウジングで UNVERIFIED が表示されてもポップアップが騒がしくなりません。

もし「知らないサイトでは必ず警告が欲しい」という場合は、未検証サイトの警告を ON にすればよいでしょう。

## カスタムリストで自分の基準を追加する

ダッシュボードの「Trust」タブでは、以下のカスタムリストを管理できます。

### JP-Anchor TLD

`.go.jp` などに加えて、自分が信頼する TLD を追加できます。追加時は先頭の `.` を含めて入力します。

### Sensitive List

金融・ゲーム・SNS 以外に、自分で警告したいドメインを登録できます。たとえば、社内ツールや個人的に注意したいサービスを追加する用途で使えます。

### Whitelist

Sensitive List に含まれていても、警告を出したくないドメインを登録できます。ホワイトリストに入ると `TRUSTED` と判定されるため、誤判定の緩和にも役立ちます。

## なぜこの設計にしたのか

ここからは、エンジニアや技術に興味のある人向けに、設計の背景を少し掘り下げます。

### プライバシーを守るためにローカル処理

Yasumaro が判定に使うリストは、すべて拡張機能内の `chrome.storage.local` に保存されます。ドメインの照合もローカルで完結し、外部 API に閲覧履歴を送ることはありません。

Tranco 更新時に `tranco-list.eu` へアクセスしますが、これは「リスト全体を取得する」ためで、個別の閲覧ドメインは送信されません。

### Bloom Filter で高速に照合

Sensitive List や Tranco List は数千〜数十万のドメインを含みます。これらを毎回線形検索すると遅くなってしまうため、Yasumaro では **Bloom Filter** を使っています。

Bloom Filter はメモリ効率の良い確率的データ構造で、候補を高速に絞り込みます。ただし、まれに偽陽性が出る可能性があります。そこで Yasumaro では、Bloom Filter で候補に絞った後に **精密照合** を行うことで、誤判定を排除しています。

### 楽観的ロックでデータの整合性を保つ

Trust Database は複数の箇所から読み書きされる可能性があります。そこで更新時には楽観的ロックを使い、同時書き込みによるデータ不整合を防いでいます。

### PermissionManager との分離

Yasumaro には「PermissionManager」という、Chrome の権限レベルでドメインを許可/拒否する仕組みもあります。TrustDb はあくまで「信頼度の評価と警告表示」に特化しており、権限管理とは責務を分離しています。

これにより、権限の問題と信頼度の問題が混在しない、保守しやすい構造になっています。

## よくある誤解

### Q. UNVERIFIED だと記録されない？

**記録されます。** UNVERIFIED はあくまで「信頼リストに含まれていない」という表示で、自動的に記録をブロックすることはありません。ただし、未検証サイトの警告を ON にしていれば、ポップアップに警告が表示されます。

### Q. SENSITIVE でも記録される？

**記録されます。** SENSITIVE は金融・ゲーム・SNS などの要注意カテゴリを示しています。警告が表示されるだけで、記録自体は止まりません。Sensitive List に入れたくないサイトがあれば、Whitelist に追加してください。

### Q. Tranco List を更新しないと何も判定されない？

**JP-Anchor TLD 判定と Sensitive List 判定は動きます。** Tranco 判定だけが無効になります。日本の公式ドメインや金融・ゲーム・SNS の警告は、更新しなくても機能します。

## 私が推奨する使い方

1. **普段は Balanced、警告は初期設定のまま**
   - 金融サイトでは警告が出るので、入力フォームのあるページでは再確認する
2. **見慣れないサービスを使うときは UNVERIFIED を意識する**
   - ただし、記録は自分で ON/OFF する
3. **Tranco List はインストール後に一度更新する**
   - これで人気ドメインが TRUSTED と判定されるようになる

## まとめ

Yasumaro のドメイン信頼度判定は、「記録を止める」ためではなく、「記録する前に少し立ち止まるきっかけ」を作る機能です。

- 4 段階の Trust Level で視覚的に分かる
- 3-Step Verification で効率的に判定
- Safety Mode や Alert Settings で自分の基準に合わせられる
- すべてローカル処理でプライバシーを守る

まだ「Tranco List を更新」していない方は、ダッシュボードの「Trust」タブから一度更新してみてください。見慣れたサイトが `TRUSTED` と表示されるようになり、逆に知らないサイトが `UNVERIFIED` と表示されることで、記録時の判断が少し楽になるはずです。
