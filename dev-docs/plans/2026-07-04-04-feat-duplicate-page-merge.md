# PBI: 類似・重複ページの自動マージ

- 親issue: [DEV-86](https://linear.app/armaniacs/issue/DEV-86)
- 領域: B. 記録品質・自動化
- type: feat / 優先度: ★推奨
- **ステータス: 実装しない（見送り）**（2026-07-05決定）

## 実装しない理由

同一日内の再訪は既存の重複チェックにより記録済みなので、それで十分と判断。
近似重複（別URLで本文がほぼ同一）の自動マージは、誤マージのリスクとコストに対して価値が見合わないため見送る。

## ユーザーストーリー

Yasumaro 利用者として、同じ記事の再訪や別URLの重複記録を自動でまとめてほしい。なぜなら、同一コンテンツが何度も記録されると履歴がノイズだらけになり、振り返りづらいから。

## ビジネス価値

履歴の S/N 比が上がり、検索・サマリ・グラフすべての品質が底上げされる。測定: マージにより削減された重複行数。

## BDD受け入れシナリオ

```gherkin
Scenario: 同一URLの再訪が集約される
  Given あるURLの履歴が既に記録されている
  When  同じURLを再訪して記録条件を満たす
  Then  新規行を増やさず既存レコードに集約される
  And   訪問回数や最終訪問時刻が更新される

Scenario: 別URLの近似重複が検知される
  Given 本文がほぼ同一で異なるURLの履歴が2件ある
  When  近似重複判定が実行される
  Then  近似重複としてまとめ、要約が統合される

Scenario: 別コンテンツは誤ってマージされない
  Given 本文が十分に異なる2件の履歴がある
  When  近似重複判定が実行される
  Then  2件は独立したまま維持される
```

## 受け入れ基準

### 単体テスト

- 集約時のフィールド更新ロジック
