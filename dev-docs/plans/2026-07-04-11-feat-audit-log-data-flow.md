# PBI: 監査ログ / データ送信の可視化

- 親issue: [DEV-86](https://linear.app/armaniacs/issue/DEV-86)
- 領域: D. プライバシー・信頼性
- type: feat / 優先度: 中

## ユーザーストーリー

プライバシーを重視する Yasumaro 利用者として、どのページの内容がどの AI プロバイダーへ送信されたかを一覧で確認したい。なぜなら、自分のデータがどこへ流れたか把握できないと安心して使えないから。

## ビジネス価値

データフローの透明性を高め、プライバシー訴求の裏付けになる。測定: 監査パネルの閲覧数 / 信頼性に関する問い合わせ低減。

## BDD受け入れシナリオ

```gherkin
Scenario: 送信イベントが監査ログに記録される
  Given full_pipeline モードでページが記録される
  When  クラウドプロバイダーへ要約リクエストが送られる
  Then  送信先プロバイダー・対象URL・日時が監査ログに残る

Scenario: ローカル完結時はクラウド送信が記録されない
  Given local_only モードでページが記録される
  When  ローカルAIのみで要約される
  Then  クラウド送信イベントは監査ログに現れない

Scenario: 監査ログをダッシュボードで閲覧できる
  Given 監査ログが蓄積されている
  When  ダッシュボードの監査パネルを開く
  Then  送信履歴が時系列で一覧表示される
```

## 受け入れ基準

- [ ] クラウド送信イベント（送信先・対象・日時）を記録する
- [ ] ローカル完結時は送信記録を残さない
- [ ] ダッシュボードで時系列一覧を表示する
- [ ] 監査ログに送信本文そのものは残さない（メタのみ）
