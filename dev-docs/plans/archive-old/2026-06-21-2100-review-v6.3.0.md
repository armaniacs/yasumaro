## 重要指摘事項（優先度順）

### [High] backupDb() の IDB/Fallback パスが JSON を Uint8Array として返す

- **指摘者:** Data Integrity Expert、API & Contract Negotiator
- **場所:** `src/offscreen/sqlite.ts:1320-1334`、`src/background/handlers/dashboardSqliteHandlers.ts:208-213`
- **影響:** OPFS パスではバイナリ .db を正しく返すが、IDB/Fallback パスでは `serialize()`（JSON）にフォールバックする。`sqliteClient.ts` の `new Uint8Array(res.data as number[])` 変換で JSON バイト列が `application/x-sqlite3` Blob としてダウンロードされ、ユーザーが破損した .db を取得する。エラーなしで誤データが渡される。
- **対処:** IDB/Fallback パスでは `{ success: false, error: 'Binary backup requires OPFS storage' }` を返し、コンシューマー（ダッシュボード）にフォールバックを明示する。`exportLogsService.ts` の `exportDb()` で null を検出し、「この環境ではバイナリエクスポートが利用できません」とユーザーに通知する。

### [High] OPFS 復旧マイグレーションのフラグクリアとデータ削除が非アトミック

- **指摘者:** Data Integrity Expert
- **場所:** `src/background/migrationService.ts:285-289`
- **影響:** `migrateOpfsRecovery()` は全バッチ成功後、まず `OPFS_FALLBACK_MODE` フラグをクリアし、次にデータを削除する。2操作の間にSW終了/クラッシュがあると、フラグ消失後に復旧不可能なオーファンデータが残る。
- **対処:** データ削除を先に実行し、最後にフラグをクリアする。または専用の完了フラグ `OPFS_RECOVERY_COMPLETED` を使用する。

### [High] `.then()` チェーンによる非同期処理（Manifest V3 ルール違反の可能性）

- **指摘者:** SRE/Ops Specialist
- **場所:** `src/background/service-worker.ts:88-100`
- **影響:** `migrationService.needsOpfsRecoveryMigration().then(...)` は V3 の推奨パターンに反し、未処理 Promise rejection のリスクがある。
- **対処:** IIFE + async/await パターンに変更する。

### [High] `consentDeclinedMessage` キーがロケールファイルに未定義

- **指摘者:** i18n Expert
- **場所:** `src/popup/privacyConsentController.ts:226-227`
- **影響:** `getMessage('consentDeclinedMessage')` はキーが存在しないため空文字を返し、ハードコードされた日本語フォールバックが常に使用される。英語ロケールでも日本語が表示され、GDPR 対応フローとして重大な i18n 欠陥。
- **対処:** 両ロケールファイルにキーを追加し、コード内のフォールバック文字列を削除する。

### [Medium] OPFS Worker handleUpdate のカラム名バリデーション欠如（Red Team + Blue Team 重複）

- **場所:** `src/offscreen/opfsWorker.ts:348-367`
- **影響:** 上流のホワイトリスト検証が存在するが、Worker 本身にはバリデーションがない。防御の深みの観点から問題。
- **対処:** `ALLOWED_COLUMNS` リストを追加し、未許可キーを無視する。

### [Medium] レガシー同意（boolean）がバージョン変更時の再同意をスキップ

- **指摘者:** Red Team Leader
- **場所:** `src/popup/privacyConsent.ts:37-40`
- **影響:** boolean `true` の同意値を持つユーザーは `needsReconsent` が設定されない。
- **対処:** boolean 型検出時に `needsReconsent: true` を設定する。

### [Medium] SQLITE_INSERT_BATCH にサイズ/件数制限なし

- **指摘者:** Blue Team Leader
- **場所:** `src/offscreen/offscreen.ts:222-240`
- **影響:** 大きなペイロードで OPFS Worker のメモリ圧迫リスク。
- **対処:** 最大件数（500件）とサマリサイズ上限を追加。

### [Medium] `query` ハンドラーが `ids` パラメータを転送しない

- **指摘者:** API & Contract Negotiator
- **場所:** `src/background/handlers/dashboardSqliteHandlers.ts:50-63`
- **影響:** ダッシュボード層から ID ベースクエリが実行できない。
- **対処:** `ids: payload.ids` を `sqliteClient.query()` に追加。

### [Medium] `orderBy` パラメータがバリデーションなし

- **指摘者:** API & Contract Negotiator
- **場所:** `src/background/handlers/dashboardSqliteHandlers.ts:58`
- **対処:** `ALLOWED_ORDER_FIELDS` でホワイトリスト検証。
