# PBI-19: 対話型設定ウィザード

## ユーザーストーリー

Yasumaro を初めて開いたユーザーとして、自分の用途に合った設定だけをステップで案内してほしい。なぜなら、設定画面の情報量が多すぎて圧倒され、初期設定で離脱するから。

## ビジネス価値

- 初期設定の離脱率を下げ、インストール→定着の漏斗を改善する
- ウィザード最後のステップ到達率で効果を測定できる
- ユーザーが必要な設定だけに集中できる

## BDD 受け入れシナリオ

```gherkin
Scenario: 初回起動時にウィザードが表示される
  Given ユーザーが Yasumaro を初めてインストールした
  And プライバシー同意フローを完了している
  When ポップアップを初めて開く
  Then 対話型設定ウィザードが表示される

Scenario: ユーザータイプ別に案内内容が変わる
  Given ウィザードが表示されている
  When ユーザーが「Obsidian 使い」を選択する
  Then Obsidian 接続設定のステップが表示される
  And AI プロバイダー設定のステップも表示される

Scenario: タイプをスキップして最小設定で完了する
  Given ウィザードが表示されている
  When ユーザーが「とりあえず試したい」を選択する
  And 「スキップ」ボタンを押す
  Then ウィザードが完了し、ダッシュボードの General セクションが開く

Scenario: 既存ユーザーにはウィザードを表示しない
  Given ユーザーが既に Yasumaro の設定を保存している
  When ポップアップを開く
  Then ウィザードは表示されず、通常のポップアップ画面が表示される
```

## 受け入れ基準

- [ ] 初回ポップアップ表示時にウィザードを表示する（プライバシー同意フローの直後）
- [ ] 3 つのユーザータイプから選択できる：
  - 「Obsidian 使い」→ Obsidian 接続設定を案内
  - 「SQLite（Obsidian なし）派」→ AI プロバイダー設定のみ案内
  - 「とりあえず試したい」→ 最小設定で完了
- [ ] 各ステップで「スキップ」ボタンを配置する
- [ ] ウィザード完了後、ダッシュボードの該当セクションが開いた状態で遷移する
- [ ] i18n（ja/en）対応
- [ ] 既存ユーザーには表示しない

## テスト戦略（t_wada スタイル）

### E2E テスト
- 初回起動からウィザード完了までのユーザージャーニー

### 統合テスト
- ウィザードのステップ遷移
- タイプ別の設定保存
- ダッシュボード遷移時のセクション開閉

### 単体テスト
- タイプ別の設定マッピング
- 既存ユーザー判定ロジック
- スキップ時のデフォルト値

## 実装アプローチ

- **Outside-In**: 統合テストからウィザードフローを定義し、失敗を確認してから実装
- **Red-Green-Refactor**: 各ステップコンポーネントを TDD で作成
- 既存の `privacyConsentController.ts` に統合し、同意後にウィザードを起動

## 見積もり

**5 ストーリーポイント**

- UI 設計・i18n: 2 SP
- ステップ遷移・設定保存: 2 SP
- 統合テスト: 1 SP

## 技術的考慮事項

- 依存関係: `privacyConsent.ts`, `getSettings()`, ダッシュボードのパネル開閉ロジック
- テスタビリティ: ウィザード状態は `chrome.storage.local` に保存し、テストでモック
- 非機能要件: アクセシビリティ（フォーカス管理、スクリーンリーダー対応）

## 実装者向け注記

### 現状コードの確認

```bash
grep -rn "privacyConsent" src/popup/
grep -rn "getSettings" src/utils/storage/
grep -rn "dashboard" src/popup/
```

既存のプライバシー同意フローが `src/popup/privacyConsentController.ts` に存在する。ウィザードは同意後のコールバックとして追加する。

### 実装手順

1. `src/popup/onboardingWizard.ts` を新規作成
2. `src/popup/popup.ts` の初期化フローにウィザード起動を追加
3. ウィザード完了フラグを `StorageKeys` に追加（例: `ONBOARDING_WIZARD_COMPLETED`）
4. ダッシュボード遷移時に該当セクションを開くパラメータを追加

### 落とし穴

- プライバシー同意が未完了のユーザーにウィザードを表示しないよう順序を制御する
- ウィザード中にブラウザを閉じた場合、再開位置を復元するかどうかを設計で決める
- 既存ユーザー誤判定を防ぐため、完了フラグはマイグレーション時に既存設定ありで ON にする
