# PBI-22: Chrome Web Store ランディングページ刷新

## ユーザーストーリー

非エンジニアの潜在的ユーザーとして、Chrome Web Store で Yasumaro を見たときに「自分のブラウジングを AI が自動整理してくれるツール」だと直感的に理解したい。

## ビジネス価値

- ストアページの表示→インストール転換率（CVR）向上
- エンジニア以外のユーザー（ビジネスパーソン、研究者、学生）への訴求力強化
- 新 UI（和モダンテーマ）を反映した最新の印象を与える

## BDD 受け入れシナリオ

```gherkin
Scenario: ストア説明文が非エンジニアにも伝わる
  Given ユーザーが Chrome Web Store の Yasumaro ページを開いた
  When 説明文を読む
  Then 「AI があなた専用のブラウジングデータベースを自動構築」という価値訴求が含まれている

Scenario: スクリーンショットが新 UI を反映している
  Given ユーザーがストアページを開いた
  When スクリーンショットを確認する
  Then 和モダンテーマ適用後のポップアップ/ダッシュボードが表示されている

Scenario: 提出用素材が整理されている
  Given 開発者がリリース作業をしている
  When `store-assets/` ディレクトリを確認する
  Then スクリーンショット、説明文、プロモーション画像がバージョン別に整理されている
```

## 受け入れ基準

- [ ] 説明文を「AIがあなた専用のブラウジングデータベースを自動構築」中心に刷新する
- [ ] スクリーンショット/GIF を新 UI（和モダンテーマ適用後）で更新する
- [ ] 必要に応じてプロモーション動画素材を用意する
- [ ] `store-assets/` ディレクトリに整理し、審査提出時の運用を明文化する
- [ ] 日英両言語のストア掲載情報を更新する

## 実装者向け注記

### 現状コードの確認

```bash
ls -la docs/index.html
ls -la scripts/build-store-zip.mjs
ls -la store-assets/ 2>/dev/null || echo "store-assets not found"
```

GitHub Pages 用ランディングは `docs/index.html` に存在。Chrome Web Store 用の掲載情報は `store-assets/` に整理する必要がある。

### 実装手順

1. `store-assets/` ディレクトリを作成
2. 説明文の日英ドラフトを作成（`store-assets/store-description.{ja,en}.md`）
3. 新 UI のスクリーンショット/GIF を作成し格納
4. 必要に応じてプロモーション動画を作成
5. `docs/STORE_ASSETS.md` に運用フローを明文化

### 落とし穴

- 説明文と実装の乖離を防ぐため、リリース時に必ず同期確認を行う
- Chrome Web Store の審査用説明文には権限の正当化を含めない（`PERMISSIONS.md` と役割を分ける）
- スクリーンショットに個人情報や機密情報が映り込まないよう注意
