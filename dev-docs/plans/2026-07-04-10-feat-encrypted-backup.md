# PBI: 暗号化バックアップのエクスポート / インポート

- 親issue: [DEV-86](https://linear.app/armaniacs/issue/DEV-86)
- 領域: D. プライバシー・信頼性
- type: feat / 優先度: ★推奨（第一弾候補）

## ユーザーストーリー

Yasumaro 利用者として、履歴と設定をパスフレーズで暗号化した1ファイルにバックアップし、別環境で復元したい。なぜなら、平文の JSON/CSV では漏洩リスクがあり、安全に持ち運べる形式が欲しいから。

## ビジネス価値

データの可搬性と安全性を両立し、機種変更・再インストール時の信頼性を高める。測定: 暗号化バックアップの作成/復元回数。

## BDD受け入れシナリオ

```gherkin
Scenario: パスフレーズで暗号化バックアップを作成する
  Given 履歴と設定が保存されている
  When  パスフレーズを指定して暗号化バックアップをエクスポートする
  Then  暗号化された単一ファイルが生成される
  And   中身は平文で読めない

Scenario: 正しいパスフレーズで復元できる
  Given 暗号化バックアップファイルと正しいパスフレーズがある
  When  インポートを実行する
  Then  履歴と設定が復元される

Scenario: 誤ったパスフレーズは復元を拒否する
  Given 暗号化バックアップファイルがある
  When  誤ったパスフレーズでインポートする
  Then  復元は失敗しエラーが表示される
  And   既存データは破壊されない
```
