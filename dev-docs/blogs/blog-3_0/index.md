# Obsidian Smart History v3.0: 世界中のユーザーに向けた信頼できる実用ツールへの進化

[ARAKI Yasuhiro ☁ ⛅🌨](https://note.com/armaniacs)

日常的にWeb閲覧をしていると、「あ、この記事あとで読みたい」「このページの内容をメモに残したい」っていう瞬間ありますよね。

Obsidian Smart Historyを使っていれば、AIが自動的に要約してくれて、Obsidianのデイリーノートに保存されます。これだけでも十分に便利ですが、ツールが成長するにつれ、「より多くの人に」「より安全に」「より快適に」使ってもらいたいという欲求が出てきました。

v3.0では、v2.3以降の内部構造を抜本的に見直し、「個人の実験ツール」から「信頼できる実用ツール」へとステップアップさせました。その進化の全貌をご紹介します。

**テーマは「国際化・セキュリティ・アクセシビリティ・パフォーマンス」。**

## Obsidian Smart History とは?

初めてこの記事をご覧になる方のために、簡単にご紹介します。

**Obsidian Smart History** は、ブラウザの閲覧履歴を **AIによる要約付きで** Obsidian のデイリーノートに自動保存する Chrome 拡張機能です。

単なる「履歴の保存」ではありません。

- 🤖 **AIによる要約**: Gemini / Groq / Ollama / OpenAI互換 など、お好みのAIで閲覧ページを自動要約
- 🎯 **スマート検出**: 5秒以上閲覧し、50%以上スクロールしたページのみを自動記録
- 🖱️ **手動記録**: 「今すぐ記録」ボタンで任意のタイミングで即座に記録も可能
- 🔒 **プライバシー保護**: 個人情報をマスク（伏せ字）してからAIに送る「Mode C」で安心
- 🌐 **多言語対応**: ブラウザの言語設定に応じて英語/日本語を自動切替

詳しい導入方法は [前回の記事 (v2.3)](https://note.com/armaniacs/n/nbb2d9b96f51c) や [GitHub リポジトリ](https://github.com/armaniacs/obsidian-smart-history) をご覧ください。

## v2.3からの進化: マスク情報の可視化

v3.0への道のりは、v2.3から始まりました。まずはv2.3で追加された機能を振り返ります。

### マスク種別表示

プライバシー保護機能（Mode C）で個人情報をマスクする際、何がマスクされたのかが一目でわかるようになりました。「電話番号3件をマスクしました」「メールアドレス1件をマスクしました」といった具体的な情報が表示されます。

### マスク箇所ナビゲーション

プレビュー画面でマスクされた箇所を簡単に確認できるようになりました。▲/▼ボタンでtextarea内の`[MASKED:*]`トークンにジャンプし、選択することができます。

### プレビューtextareaのリサイズ

プレビュー画面のtextareaを右下ハンドルで縦横自由にリサイズできるようになりました。長いテキストも快適に確認できます。

## 進化1: 「世界中のユーザー」へ - 完全多言語対応 (i18n)

これまでの v2.3 までは、UIやエラーメッセージが日本語にハードコードされていました。これでは日本語以外の環境では使いにくいものでした。

v2.4以降では、**Chrome拡張機能の標準的な多言語化（i18n）** を完全に実装しました。

### 何が変わったのか?

- **自動言語切替**: ブラウザの設定が英語なら英語で、日本語なら日本語で表示されます。
- **エラーメッセージのローカライズ**: 「保存に失敗しました」から「Save failed」まで、すべてのメッセージが翻訳されています。
- **動的なメッセージ**: 「電話番号3件をマスクしました」といった、数値を含むメッセージも自然な多言語対応 (`Masked 3 phone numbers`) を実現しました。

### UIのアイコン化

多言語化すると、ボタンのテキスト（「設定」と「Settings」）によってレイアウトが崩れがちです。そこで、設定ボタンなどを直感的なアイコン（⚙ ☰ ← ×）に変更しました。あわせて、スクリーンリーダー用の `aria-label` を適切に設定し、アクセシビリティも確保しています。

開発の裏側では、`_locales/en/messages.json` と `_locales/ja/messages.json` にすべての文言を切り出し、コード内からハードコードされた日本語をすべて一掃しました。これにより、今後さらに多くの言語への対応も容易になっています。

## 進化2: 見えないところで「鉄壁の守り」を - セキュリティ強化

ブラウザ拡張機能は、ユーザーの閲覧情報という極めて機密性の高いデータにアクセスします。だからこそ、最高水準のセキュリティが求められます。v2.4以降では、セキュリティ専門家の視点から「徹底した守り」を実装しました。

セキュリティは目に見えない部分ですが、筆者としてはこの見えない守りにこそ一番力を入れました。

### 1. 内部ネットワークの保護 (SSRF対策)

uBlockフィルタの外部リストを読み込む際、悪意のあるURLを指定されることで、自宅や会社の「内部ネットワーク」にアクセスされるリスク（SSRF）がありました。

- **対策**: `isPrivateIpAddress()` 関数を実装。192.168.x.x や localhost へのフィルター取得を厳格にブロックし、内部ネットワークを保護しました。

### 2. スクリプト注入の排除 (XSS対策)

ユーザー入力やAIの回答を画面に表示する際、悪意のあるスクリプトが紛れ込むリスク（XSS）を排除しました。

- **対策**: `innerHTML` の使用を完全に撤廃。すべて `textContent` と `createElement` で組み立てるセキュアな実装に変更しました。

### 3. 機密情報の漏洩防止

万が一エラーが発生した際、エラーメッセージに「APIキー」や「Bearerトークン」が含まれてしまうことがあります。

- **対策**: `sanitizeErrorMessage()` を実装。エラーをユーザーに表示する前に、機密情報を自動的に `[HIDDEN]` に置き換える処理を追加しました。

### 4. APIキーの自動暗号化 (v3.0の新機能)

v3.0では、APIキーの保護をさらに強化しました。

- **AES-GCM暗号化**: `chrome.storage.local` に保存されるAPIキーをAES-GCMで自動暗号化
- **PBKDF2キー導出**: 100,000イテレーションのキー導出で、総当たり攻撃に耐性
- **後方互換性**: 既存の平文APIキーはそのまま読み取り可能（自動マイグレーション）
- **透過的な動作**: 呼び出し元（popup.js, aiClient.js, obsidianClient.js）の変更不要

### 5. 動的URL検証の実装 (v3.0の新機能)

ユーザーが設定したURLのみにアクセスを制限する機能を追加しました。

- **URL正規化**: プロトコル、パス、クエリ・フラグメントの正規化
- **許可リスト管理**: ユーザーが設定したURLのみにアクセスを制限
- **ハッシュベースの検証**: URLリストのハッシュを計算し、変更検出

### 6. 最小権限の原則 (Least Privilege)

これまでは `<all_urls>` という広範な権限を求めていましたが、これを `http://*/*` と `https://*/*` に限定しました。ブラウザの管理画面（chrome://）やローカルファイル（file://）への不要なアクセスを防止しています。

### その他の多層防御

上記に加え、以下のような防御策も実装しています。

- **Content Security Policy (CSP)**: スクリプトインジェクションのリスクを軽減
- **URLパスサニタイズ**: パストラバーサル攻撃（`../` を使ったディレクトリ侵入）を防止
- **クリックジャッキング対策**: iframe埋め込みによる攻撃を防止
- **ポート番号検証**: 不正な入力値を拒否
- **メモリ枯渇防止**: URLセットのサイズ制限（最大10,000件）

これらの対策は、825件超のテスト（XSS、SSRF攻撃パターンを含む）で検証されています。

## 進化3: 「誰もが迷わず使える」へ - アクセシビリティの飛躍

ツールは「使える」だけでなく、「心地よく使える」必要があります。特にキーボード操作の快適さとスクリーンリーダーへの対応を追求しました。

### 矢印キーで設定タブを切り替え

マウスに持ち替えなくても、矢印キー（←→）だけで「全般」「フィルタ」「プライバシー」といった設定タブを切り替えられるようになりました。Home/Endキーで先頭/末尾のタブにジャンプすることもできます。

### フォーカストラップとモーダル操作

確認画面（モーダル）が表示された際、Tabキーを押してもフォーカスがモーダルの外に飛び出しません。Tabキーでモーダル内のボタン間を循環し、ESCキー一発で閉じることができます。

### スクリーンリーダー対応

`role="tab"` / `role="dialog"` / `aria-selected` / `aria-modal` といったARIA属性を適切に付与し、スクリーンリーダーでも設定画面の構造が正しく伝わるようになりました。

### タッチ領域の確保

アイコンボタンのサイズを44×44pxに拡大し、WCAG（Webアクセシビリティガイドライン）で推奨されるタッチ領域を確保しました。すべてのフォーカス可能な要素に視覚的なフォーカスインジケーターも追加しています。

### ダークモード対応 (v3.0の新機能)

OS設定に連動する自動ダークテーマを追加しました。

- **自動切替**: `@media (prefers-color-scheme: dark)` でOS設定に連動
- **コントラスト改善**: ダークモードでもWCAG AA準拠のコントラスト比を確保
- **ネイティブ要素対応**: `color-scheme: dark` でブラウザネイティブ要素も暗色化

## 進化4: 「より快適に」へ - パフォーマンス改善

v3.0では、パフォーマンスの面でも大幅な改善を行いました。

### PIIサニタイザのパフォーマンス改善

個人情報マスク処理のアルゴリズムをO(n²)からO(n)へ最適化しました。

- **Setを使用した重複チェック**: O(1)探索で高速化
- **パターンオーバーラップ修正**: より具体的なパターン（クレジットカード）を優先
- **マッチ位置のソート**: 長さ降順でソートし、オーバーラップ防止

### URLセットLRU排除機能

古いURLの自動クリーンアップ機能を実装しました。

- **タイムスタンプベースのLRU管理**: MAX_URL_SET_SIZE: 10,000
- **しきい値超過時の自動削除**: 最古のURLを自動削除

### Service Worker初期化の遅延化

不要なタブクエリをスキップし、初期化を遅延化しました。

- **TabCacheの遅延初期化**: 必要になるまで初期化を遅延
- **全タブの初期化を回避**: 必要なタブIDのみを直接操作

## 進化5: 「より美しく」へ - UI/UXの改善

v3.0では、UI/UXの面でも細かく手を入れています。

### コントラスト比の改善

WCAG AA準拠のためにテキスト・ボタンの色を濃くしました。

- ラベルテキスト: `#555` → `#333`
- プライマリーボタン・confirmボタン: `#4CAF50` → `#2E7D32`
- ヘルプテキスト: `#666` → `#444`

### インラインバリデーションとアクセシビリティ強化

入力フィールドにリアルタイムエラー表示を追加しました。

- Protocol, Port, Min Visit Duration, Min Scroll Depth に `aria-invalid` と `aria-describedby` 属性を追加
- 各フィールドに `.field-error` エラーメッセージ表示エリア（`role="alert"`）を追加
- `blur` イベントでリアルタイムバリデーション、保存時に一括エラー表示

### CSSカラーパレットの一貫性化

ハードコード色をCSS変数に統一しました。

- `:root` に50以上のCSS変数を定義（Primary, Accent, Secondary, Danger, Success, Info, Text, Borders）
- `styles.css` 全体でハードコード色を `var()` に置換し、テーマ変更を容易に

### モーダル・ドロップダウンのトランジション追加

表示切替にアニメーションを実装しました。

- モーダル: `opacity` + `translateY` の0.2sトランジション（`.show`クラス制御）
- ドロップダウンメニュー: `opacity` + `translateY` の0.15sトランジション

## 進化6: 「シンプルに」へ - TypeScript完全除去

v2.4.6で、プロジェクトからTypeScriptを完全に除去し、プレーンJavaScriptのみのChrome拡張に移行しました。

- `.ts` ファイル27個を削除（対応する `.js` ファイルは全て存在済み）
- TypeScript設定ファイルを削除（`tsconfig.json`, `tsconfig.build.json`, `.tsbuildinfo`）
- `package.json` からTS関連のscriptsとdevDependenciesを除去
- `jest.config.cjs` から `ts-jest` transform設定と `.ts` テストパターンを除去
- 不要な `dist/` ディレクトリを削除

これにより、ビルド構成がシンプル化され、メンテナンス性が向上しました。

## インストール・更新方法

### すでに導入している方

GitHubから最新のソースコードを取得し、`chrome://extensions` で「再読み込み」をクリックしてください。設定は自動的に引き継がれます。

### 初めて導入する方

1. **Obsidian** 側で [Local REST API プラグイン](https://github.com/coddingtonbear/obsidian-local-rest-api) をインストール・有効化します。
2. **AIの準備**: [Google Gemini API](https://aistudio.google.com/app/apikey)（無料枠あり）や [Groq](https://console.groq.com/keys)（無料枠あり・高速でおすすめ）の APIキーを用意します。
3. [GitHub](https://github.com/armaniacs/obsidian-smart-history) から ZIP をダウンロードし、解凍します。
4. Chrome の拡張機能画面（`chrome://extensions`）で「デベロッパーモード」をONにし、「パッケージ化されていない拡張機能を読み込む」で解凍したフォルダを選択します。

設定画面で Obsidian API Key と AIプロバイダーを設定すれば、すぐに使い始められます。
