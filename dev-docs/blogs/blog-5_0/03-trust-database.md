# 「このサイト、記録していいんだっけ」という問いに答える仕組み

ネットバンキング、SNS、ゲームのアカウントページ。これらのページを眺めていると、あることが気になるようになりました。

Obsidian Weaveはページの内容をAIに要約させてObsidianに保存します。それ自体は便利なのですが、「このサイトって記録してよかったんだっけ」という迷いが毎回発生します。金融サイトのページはCleansingで入力フォームを取り除いているとはいえ、記録そのものをしていいのか。ニュースサイトと同じ感覚で記録してしまっていいものかどうか。

この感覚に対処するために、v5.0 でドメイン信頼度判定システム（Trust Database）を実装しました。

---

## ドメインを3段階で評価する

Trust Database は、アクセス中のドメインを次の3レベルに分類します。

| レベル | 意味 | 例 |
|--------|------|---|
| `TRUSTED` | 信頼済みの情報源 | `.go.jp`、`.ac.jp`、Tranco上位サイト |
| `SENSITIVE` | 機密性の高いサービス | 銀行、SNS、ゲームアカウント |
| `UNVERIFIED` | 判定不能 | 未登録・低トラフィックのサイト |

これは「ブロック」や「許可」の仕組みではありません。「このサイトがどんな性格を持つか」を記録前に把握するための情報です。

---

## 3-Step Verification の仕組み

判定は3つのステップで行われます。速い順に評価し、最初に一致したレベルが採用されます。

```
訪問ドメイン
      ↓
1. JP-Anchor TLD確認（.go.jp / .ac.jp / .lg.jp 等）
   → 一致すれば TRUSTED
      ↓
2. Sensitive リスト確認（金融 / SNS / ゲーム）
   → 一致すれば SENSITIVE
      ↓
3. Tranco ランキング確認（上位N件の信頼済みドメイン）
   → 上位なら TRUSTED / それ以外は UNVERIFIED
```

ステップ1の JP-Anchor は `.go.jp`（政府機関）、`.ac.jp`（学術機関）、`.lg.jp`（地方公共団体）のような公的TLDが対象です。設定画面からユーザーが自分で追加することもできます。

ステップ3で使う Tranco リストは、Webトラフィックの研究目的で公開されているドメインランキングです。「大手サービスかどうか」の目安として使っています。

---

## Bloom Filter による高速照合

Tranco リストは最大10万件のドメインを含みます。毎回フルスキャンするのは遅いため、Bloom Filter を使っています。

Bloom Filter はドメインが「おそらくリストに含まれる」かどうかを高速に確認できるデータ構造です。偽陽性（実際にはない→あると判定）が約1%発生しますが、そのあと精密照合で弾きます。偽陰性（実際にある→ないと判定）は発生しません。

```
Tranco 10,000件 → Bloom Filter化（約64KB）
                           ↓
         bloomFilter.mightContain(domain)
           ↓ Yes               ↓ No
         精密照合            UNVERIFIED 確定
           ↓ 一致
         TRUSTED 確定
```

データは `chrome.storage.local` に保存されます。拡張機能起動時に復元されるため、毎回ネットワーク通信は発生しません。

### サブドメインの扱い

`edition.cnn.com` にアクセスしている場合、Tranco リストには `cnn.com` として登録されています。このため照合は段階的に行われます。

```
edition.cnn.com → マッチしない
      ↓
   cnn.com     → マッチ → TRUSTED
```

`www.` などのサブドメインがあっても、ルートドメインが Tranco に含まれていれば TRUSTED と判定されます。

## Sensitive リストの構成

Sensitiveリストには3カテゴリのプリセットが内蔵されています。

| カテゴリ | 内容 |
|---------|------|
| finance | 国内主要銀行・カード会社 20ドメイン |
| gaming | 任天堂・バンナム・スクエニなど 10ドメイン |
| sns | Twitter(X)・Instagram・LINEなど 7ドメイン |

これらはソースコードに固定されているプリセットです。設定画面から個別に追加・除外も可能です。

「このプリセットに入っているが、自分にとっては普通のサイトだ」という場合は、ホワイトリストに追加することでSENSITIVE判定を除外できます。

---

## Tranco リストの更新

Tranco リストはデフォルトでは空です。必要に応じてダッシュボードの Trust パネルから手動で取得します。

```
Tranco Trust Tier:  [Top 10,000 ▼]
                    [Tranco Listを更新]
ドメイン数: 10000 | Tier: Top 10,000 | 最終更新: 2026-03-14
```

Tier は3段階から選べます。

| Tier | 件数 | 適する用途 |
|------|------|-----------|
| Top 1,000 | 1,000件 | 有名サービスだけを信頼済みにしたい |
| Top 10,000 | 10,000件 | デフォルト。バランス重視 |
| Top 100,000 | 100,000件 | より広くTRUSTEDにしたい |

Safety Mode（strict / balanced / relaxed）はこのTierと連動しています。Safety Modeを変えるとTierも自動的に切り替わります。

更新時は Tranco API から最新のリストIDを取得してからCSVをダウンロードします。データは `chrome.storage.local` に保存されるため、2回目以降の起動時にネットワーク通信は発生しません。

更新の頻度はTrancoの公式推奨にあわせて週1〜月1回程度で十分です。毎日更新する必要はありません。

---

## JP-Anchor と Tranco の違い

Q. `.go.jp` はTRUSTEDになるのに、有名な民間企業のサイトはUNVERIFIEDになることがあるのはなぜですか？

A. JP-Anchor はTLDで機械的に判定するため、`.go.jp` ならすべてTRUSTEDになります。Tranco は取得・更新が必要なため、更新前は空のままです。「Tranco を更新していないから UNVERIFIED になる」ケースは多いです。一度更新してしまえば以降は記録されます。

---

## Phase 2: 記録時の警告表示

Phase 1 が判定ロジックと設定UIの土台だとすれば、Phase 2 はそれを実際の記録フローに接続する段階です。

実装されたのは **Alert Settings** です。カテゴリごとに警告ON/OFFを個別制御できます。

| 設定項目 | デフォルト | 意味 |
|---------|-----------|------|
| 金融サイトの警告 | ON | finance カテゴリで警告バッジを表示 |
| 警戒ドメインの警告 | ON | sensitive リストにあるドメインで警告 |
| 未検証サイトの警告 | OFF | UNVERIFIED ドメインでも警告したい場合に有効化 |

警告はモーダルではなく、ポップアップのステータスパネルに **Trust バッジ**として表示されます。記録そのものをブロックするのではなく、「このサイトはこういう性格です」とユーザーに知らせる設計です。

また「警告されたページも記録に残す」オプションを有効にすると、警告が出た場合でも記録が中断されず履歴に残ります。

---

## 設定の場所

ダッシュボードの「Trust」タブから設定できます。

- **Safety Mode**: strict / balanced / relaxed を選択
- **JP-Anchor TLD管理**: カスタムTLDを追加・削除
- **Sensitive ドメイン管理**: finance / gaming / sns カテゴリごとに追加
- **ホワイトリスト**: SENSITIVEから除外するドメインを登録
- **Tranco更新**: Tier選択 → Update ボタンで取得

はじめのうちは「Tranco を一度更新してみる」だけで十分です。金融サイトをよく記録する方は、ホワイトリストに登録してしまうのが手軽です。
