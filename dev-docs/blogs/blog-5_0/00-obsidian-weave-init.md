「あのページ、どこで読んだっけ」という経験は誰にでもあるはずです。

ブックマークに入れても見返さない。メモに書き写す手間はかけたくない。でも、読んだという事実と内容の断片は残しておきたい。

**Obsidian Weave** は、そのギャップを埋るために作った Chrome 拡張機能です。ブラウザで読んだページを、AIが要約してObsidianのデイリーノートに自動保存します。

---

## Obsidian Weave とは

Obsidian Weave は、**ブラウザの閲覧履歴を AI 要約付きで Obsidian のデイリーノートに自動保存する Chrome 拡張機能**です。

単なる「履歴の保存」ではありません。ページを読んでいる間に、バックグラウンドで以下が行われます。

1. 「このページ、ちゃんと読んだ」と判断されると自動で記録が走る
2. ページ本文が AI に送られ、1〜2 文の要約が生成される
3. Obsidian のデイリーノートの「ブラウザ閲覧履歴」セクションに追記される

```
## ブラウザ閲覧履歴

- 14:32 [TypeScriptのbranded typesについて](https://example.com/ts-branded)
    - #IT・プログラミング TypeScriptにおけるbranded typeパターンの解説。型の安全性を高めるためにプリミティブ型をラップする手法と、実際のユースケースを紹介。
```

Obsidian を開かなくても、読むだけで記録が溜まっていきます。

---

## 「読んだ」と判断する仕組み

ページを開いただけで全部記録されたら、それはノイズにしかなりません。

Obsidian Weave は **5 秒以上閲覧 + 50% 以上スクロール** という条件を満たしたページだけを自動記録します。この閾値はダッシュボードで変更できます。

「今すぐ記録」ボタンで任意のページを即座に記録することも可能です。

## セットアップ

必要なものは 3 つです。

### 1. Obsidian + Local REST API プラグイン

Obsidian に [Local REST API プラグイン](https://github.com/coddingtonbear/obsidian-local-rest-api) をインストールして有効化します。プラグインの設定画面に表示される API キーをコピーしておいてください。

### 2. AI プロバイダー

以下のいずれかを用意します。**Groq は無料枠があり、応答も速いので最初の選択肢としておすすめです。**

| プロバイダー | 特徴 |
|------------|------|
| [Groq](https://console.groq.com/keys) | 無料枠あり・高速・推奨 |
| [Google Gemini](https://aistudio.google.com/app/apikey) | 無料枠あり |
| [OpenAI](https://platform.openai.com/api-keys) | 安定した品質 |
| [Anthropic (Claude)](https://console.anthropic.com/) | 高品質な要約 |
| Ollama / LM Studio | APIキー不要・完全ローカル |

### 3. 拡張機能のインストール

```bash
git clone https://github.com/armaniacs/obsidian-weave.git
cd obsidian-weave
npm install
npm run build
```

Chrome の `chrome://extensions` を開き、「デベロッパーモード」を有効にして「パッケージ化されていない拡張機能を読み込む」から `dist` フォルダを選択します。

### 設定

ツールバーの Obsidian Weave アイコンをクリック → ⚙ からダッシュボードを開き、以下を設定します。

- **Obsidian API キー**: Local REST API プラグインのキー
- **AI プロバイダー**: 使いたいプロバイダーを選択し、API キーを入力
- **デイリーノートのパス**: Obsidian のデイリーノートが保存されている場所

「保存 & 接続テスト」を押して両方 OK になれば完了です。

## 主な機能

### 自動記録 + 手動記録

ページを十分に読めば自動で記録されます。「今すぐ記録」ボタンで強制記録も可能。記録されたときはツールバーアイコンに青い `◎` バッジが表示されるので、「保存されたかな」と確認しなくて済みます。

### ドメインフィルター

「このサイトは記録しない」という設定ができます。ホワイトリスト/ブラックリスト形式で、ワイルドカードパターンも使えます。uBlock Origin 形式のフィルターリストを直接インポートすることも可能です。

### AI プロンプトのカスタマイズ

要約の形式を変えたい場合は、ダッシュボードの「プロンプト」パネルからカスタマイズできます。5 種類のプリセットが用意されています。

| プリセット | 出力形式 |
|-----------|---------|
| デフォルト | 日本語で 1〜2 文の簡潔な要約 |
| タグ付き要約 | `#カテゴリ` タグ + 1 行要約 |
| 箇条書き | 日本語で 3 点の箇条書き |
| 英語要約 | 英語で 1〜2 文の要約 |
| 技術的観点 | 技術的なポイントを日本語で 3 点 |

プリセットを複製してカスタマイズすることもできます。

### タグ付き要約モード

「タグ付き要約」プリセットを使うと、AI が自動でカテゴリタグを付けてくれます。

```
- 09:25 [記事タイトル](https://example.com)
    - #IT・プログラミング #インフラ・ネットワーク ここにAI要約が入ります。
```

Obsidian のタグ検索で過去の記録を絞り込めます。記録直後にポップアップでタグが確認できるので、Obsidian を開かなくても分類結果がわかります。

### プライバシー保護

4 つのプライバシーモードから選択でき、メールアドレス・電話番号・クレジットカード番号などをマスク（伏せ字）してから AI に送信できます。ネットバンキングのような機密性の高いページでは、自動でプライバシーヘッダーを検出して記録を止める仕組みもあります（ツールバーにオレンジの `!` バッジが表示されます）。

### ローカル LLM 対応

**LM Studio** や **Ollama** を使えば、API 料金なし・データがクラウドに送られない完全ローカル環境で要約できます。

```bash
# Ollama の場合（CORS 設定が必要）
OLLAMA_ORIGINS=chrome-extension://* ollama serve
```

ダッシュボード → AI プロバイダーで「Ollama」または「LM Studio」を選択し、プリセットボタンで Base URL を自動入力できます。3B 規模の小型モデルでも実用的な要約が得られます。

## v5.0 で追加されたプライバシー・信頼性機能

### サイトごとの記録許可

v5.0 からは **サイトごとに明示的な許可が必要** になりました。未許可のサイトでは `🔒 LOCKED` バッジが表示され、「このサイトを許可する」ボタンで有効化します。Google・GitHub・Zenn など Tranco Top 1000 に含まれる主要サイトは最初から許可済みです。

### ドメイン信頼度インジケーター

ポップアップに信頼度が表示されます。

| 表示 | 意味 |
|------|------|
| ✅ TRUSTED | Tranco 上位ランク or JP 公的ドメイン（.go.jp 等） |
| ⚠️ SENSITIVE | 金融・ゲーム・SNS など注意が必要なカテゴリ |
| ❓ UNVERIFIED | ランク外・未検証のドメイン |

「このサイト、記録していいんだっけ」という迷いに答えるための情報です。記録そのものをブロックするのではなく、判断材料を提供します。

### Content Cleansing（コンテンツクレンジング）

ネットバンキングや SaaS のページには、フォーム・残高表示・スクリプトなど AI に送りたくない情報が大量に含まれています。**Content Cleansing** を有効にすると、記録前にこれらを自動削除します。

- **Hard Strip**: `<input>`、`<script>`、`<iframe>` など機密・非コンテンツ要素を削除
- **Keyword Strip**: `balance`、`account`、`meisai` などのキーワードを含む要素を削除

ポップアップの「詳細を表示」から、記録ボタンを押す前に削除件数を確認できます。デフォルトはオフ。ダッシュボードの「Content Cleansing」パネルで設定します。

## よくある使い方

**技術記事を読んだ日の Obsidian ノート例:**

```
## ブラウザ閲覧履歴

- 09:12 [Rust の所有権システム入門](https://example.com/rust-ownership)
    - #IT・プログラミング Rustの所有権・借用・ライフタイムの基本概念を図解で解説。
- 10:34 [PostgreSQL のインデックス戦略](https://example.com/pg-index)
    - #IT・プログラミング #インフラ・ネットワーク B-treeとGINインデックスの使い分けと、EXPLAINでのチューニング方法を実例付きで解説。
- 14:07 [スタートアップのプロダクト戦略](https://example.com/startup)
    - #ビジネス・経済 PMFに至るまでの仮説検証サイクルと、成長フェーズ別の戦略転換点をまとめた記事。
```

読むだけでこれが溜まっていきます。

## 設定のすすめ

最初にやっておくと快適になる設定です。

1. **プロンプトを「タグ付き要約」に変更**: Obsidian のタグ検索と組み合わせると強力
2. **Tranco リストを更新**: ダッシュボード → Trust → 「Tranco List を更新」。これで主要サービスの信頼度判定が機能するようになります
3. **記録しないドメインを設定**: 社内システムやプライベートなサイトはブラックリストに追加
