# Obsidian Weave v5.0 — ユーザー向け新機能ガイド

> 対象読者: Obsidian Weave を日常的に使っているユーザー


---

## v5.0 の主な変更点

### 1. 自動保存されたらバッジが光る 🔵

ページが自動記録されると、Chrome ツールバーのアイコンに **青色の `◎` バッジ**が表示されます。「あのページ、ちゃんと保存されたかな？」という確認が不要になりました。

- 同じタブにいる間はバッジが継続表示されます
- 別のページへ移動すると自動でバッジがクリアされます

---

### 2. サイトごとに記録を許可する仕組みに 🔒

これまではインストール時に全サイトへのアクセスを一括で許可していました。v5.0 からは**サイトごとに明示的な許可**が必要になります。

- 未許可のサイトでは **🔒 LOCKED** バッジが表示されます
- 「このサイトを許可する」ボタンをクリックするだけで記録が有効になります
- ダッシュボードの「権限提案リスト」で、よく訪問しているが未許可のサイトを一括確認・許可できます

よく使うサイト（Google、GitHub、Zenn など Tranco Top 1000）はあらかじめ許可済みです。

---

### 3. ドメインの信頼度が3段階で表示される 🛡️

ポップアップに**信頼度インジケーター**が追加されました。

| 表示 | 意味 |
|------|------|
| ✅ TRUSTED | Tranco 上位ランク or JP 公的ドメイン |
| ⚠️ SENSITIVE | 金融・ゲーム・SNS など注意が必要なカテゴリ |
| ❓ UNVERIFIED | ランク外・未検証のドメイン |
| 🔒 LOCKED | 記録権限が未付与 |

ダッシュボードの「Trust」パネルから、Safety Mode（strict / balanced / relaxed）や Tranco Tier（Top 1k / 10k / 100k）を調整できます。

---

### 4. 記録前にページを「掃除」できる — Content Cleansing 🧹

ネットバンキングや SaaS のページには、フォーム・残高表示・スクリプトタグなど、AIに送りたくない情報が大量に含まれています。**Content Cleansing** を有効にすると、記録前にこれらを自動削除します。

削除方法は2種類です：

- **Hard Strip**: `<input>`、`<script>`、`<iframe>` など機密・非コンテンツ要素をタグ単位で削除
- **Keyword Strip**: `balance`、`account`、`meisai` など指定キーワードを含む要素を削除

ポップアップの「詳細を表示」から、記録ボタンを押す前に削除件数を確認できます。ダッシュボードの「Content Cleansing」パネルで設定します（デフォルト: オフ）。

> 関連記事: [AIに送る前にページを「掃除」する](./cleansing-all.md)

---

### 5. AI要約のノイズをさらに削減 — AI Summary Cleansing / Deep Cleansing 📉

Content Cleansing に加えて、**AI Summary Cleansing** は AI へ送るコンテンツからナビゲーション・広告・ソーシャルウィジェットを除去します。さらに実験的な **Deep Cleansing** を有効にすると、非表示要素・空要素・リンク密度の高いリストも削除対象になります。

ダッシュボードの履歴パネルで各エントリを展開すると、クレンジング前後のバイト数削減率が確認できます。

> 関連記事: [AIに送る前にページを「掃除」する](./cleansing-all.md)

---

### 6. 保存直後にAIタグが表示される 🏷️

タグ付き要約モードを使っている場合、「今すぐ記録」ボタンを押した瞬間にポップアップで AI が付けたタグが確認できます：

```
✓ Obsidianに保存しました (2.2秒)
🏷 AIタグ: #IT・プログラミング  #ビジネス・経済
```

Obsidian を開かなくても分類結果をその場で確認できます。タグが表示される場合、ポップアップの自動クローズタイマーが 4 秒に延長されます。

---

### 7. AI要約のトークン数・バイト数が見える 📊

ダッシュボードの履歴パネルで、各ページの記録エントリを展開すると以下が確認できます：

- **AI要約**: AIが生成した要約テキスト
- **トークン数・処理時間・AI情報**: 例: `トークン数: 送信: 7,398, 受信: 606, 処理時間 3.1秒 (AI: openai / gpt-4o-mini)`
- **クレンジング削減率**: 元のバイト数 → 送信バイト数の削減量
- **AI要約クレンジング理由**: 複数種類が削除された場合は上位3件を表示（例: `ナビゲーション, 広告, ソーシャル`）

AIへ送るコンテンツが事前にクレンジング（不要なナビゲーション・広告要素を除去）されていることが確認できます。

### 8. ローカルLLM対応（LM Studio / Ollama）🖥️

**LM Studio** と **Ollama** をAIプロバイダーとして使えるようになりました。API料金なし・データがクラウドに送られない完全ローカル環境で要約できます。

ダッシュボード → 初期設定 → AI プロバイダーで選択してください。

---

#### LM Studio

1. LM Studio を起動し、左サイドバーの「Developer」→「Start Server」をオンにする
2. AIプロバイダーで「**LM Studio**」を選択
3. 「LM Studio」プリセットボタンで Base URL（`http://localhost:1234/v1`）が自動入力されます
4. 読み込んでいるモデル名を入力（省略すると自動検出）
5. 「AIテスト」で接続確認

**動作確認済みモデル**: `gemma-4-e2b-it`（Google Gemma 4、2B）

出力例:

```
- 09:41 [インターステラテクノロジズの小型ロケットMOMOの先にあるもの | 宙畑](https://sorabatake.jp/691/)
    - #IT・プログラミング #インフラ・ネットワーク インターステラテクノロジズ社による小型ロケット開発と宇宙ビジネスへの参入に関する技術的な試み（MOMO2号機、MOMO3号機打ち上げ）の結果と、大型ロケットと小型ロケットの輸送手段としてのビジネスモデル（路線バスとタクシーの例え）について分析している。また、100kg超小型衛星の宇宙輸送におけるコストと柔軟性のトレードオフ、および関連する企業（Virgin Orbit、Rocket Labなど）の動向が考察されている。
```

---

#### Ollama

1. Ollama をインストールし、使いたいモデルを取得する
   ```bash
   brew install ollama
   ollama pull llama3.2   # 使いたいモデルを指定
   ```
2. **CORS設定をしてから**サーバーを起動する（これをしないと403エラーが出ます）
   ```bash
   OLLAMA_ORIGINS=chrome-extension://* ollama serve
   ```
3. AIプロバイダーで「**Ollama**」を選択
4. 「Ollama」プリセットボタンで Base URL（`http://localhost:11434/v1`）が自動入力されます
5. モデル名を入力（例: `llama3.2`、`mistral`）
6. 「AIテスト」で接続確認

> **macOS で常時起動する場合（launchd）:**
> ```bash
> launchctl setenv OLLAMA_ORIGINS "chrome-extension://*"
> ```
> この設定をしてから Ollama を再起動してください。

**動作確認済みモデル**: `MHKetbi/sbintuitions-sarashina2.2-3b-instruct-v0.1:q4_K_S`（3B・日本語特化）

出力例:

```
- 19:09 [インターステラテクノロジズの小型ロケットMOMOの先にあるもの | 宙畑](https://sorabatake.jp/691/)
    - #IT・プログラミング #ビジネス・経済 インターステラテクノロジズは小型ロケットMOMOの開発を進めており、堀江貴文氏が率いる同社は宇宙ビジネスに参入している。MOMO2号機の打ち上げは失敗したが、3号機は成功し、高度100kmに到達した。しかし、ロケット打ち上げの成功だけでは売上にはつながらず、衛星を宇宙に運ぶ運送屋として、小型衛星の需要や費用の差が小型ロケットビジネスの鍵となる。
```

---

#### さくらのAIクラウド（OpenAI互換API）

さくらインターネットが提供する国産AIクラウドサービスで、OpenAI互換APIとして利用できます。データが国内に留まる点が特徴です。

1. さくらのAIクラウドでAPIキーとエンドポイントURLを取得する
2. AIプロバイダーで「**OpenAI Compatible**」を選択
3. Base URL・APIキー・モデル名を入力
4. 「AIテスト」で接続確認

**動作確認済みモデル**: `gpt-oss-120b`（さくらのAIエンジン提供）

出力例:

```
- 19:29 [インターステラテクノロジズの小型ロケットMOMOの先にあるもの | 宙畑](https://sorabatake.jp/691/)
    - #ビジネス・経済 #インフラ・ネットワーク インターステラテクノロジズの小型ロケットMOMOシリーズは、MOMO2号機の失敗とMOMO3号機の成功を経て、衛星輸送を「タクシー」的サービスと位置付け、打ち上げ頻度や柔軟性は大型ロケットに優れるもののコストは倍以上と課題があることを示す。小型ロケット市場は多数のベンチャーが参入し、顧客獲得と価格競争が鍵となるが、政府支援やクラウドファンディングなどの資金調達が事業推進に重要な役割を果たしている。
```

---

#### モデルによる出力の違い

同じ記事・同じプロンプトで3モデルを比較すると、スタイルの違いが明確に出ます。

| モデル | 環境 | パラメータ規模 | 特徴 |
|--------|------|--------------|------|
| Sarashina 2.2 3B | Ollama（ローカル） | 3B・日本語特化 | 固有名詞を正確に捉え、事実を簡潔にまとめる |
| Gemma 4 2B | LM Studio（ローカル） | 2B | 分析的。ビジネスモデルの例えや競合企業まで言及 |
| gpt-oss-120B | さくらのAIクラウド | 120B | 構造化された2文構成。成功/失敗→課題→市場環境と論理的に展開 |

**考察**:

- **3B規模のローカルモデル**でも、タグ付きで実用的な要約が得られる。ただし出力の安定性はモデルによってばらつきがある
- **Sarashina**は日本語特化の効果が出ており、固有名詞（堀江貴文氏）や文脈（高度100km到達）を正確に拾っている
- **Gemma 4 2B**はパラメータ数が最小ながら分析の深さが際立つ。路線バス/タクシーの比喩やRocket Lab等の競合言及はプロンプトの指示を超えた推論といえる
- **120Bのクラウドモデル**は情報の取捨選択と構造化が洗練されている。「タクシー的サービス」という核心を1文目で掴み、2文目で市場課題を展開する構成は再現性が高い

**応答速度**（M1 MacBook での実測値）:

| モデル | 環境 | 応答時間 |
|--------|------|---------|
| gpt-oss-120B | さくらのAIクラウド | 3〜5秒（安定） |
| gemma-4-e2b-it | LM Studio（ローカル） | 約19秒 |
| Sarashina 2.2 3B | Ollama（ローカル） | 約46秒 |

ローカルモデルはプライバシーとコストの面で優れ、大規模クラウドモデルは品質の安定性と応答速度で優れます。用途・予算・データ管理ポリシーに応じて選択してください。なおローカルモデルを使う際はダッシュボードのAIタイムアウトを120秒以上に設定しておくことを推奨します。モデルの切り替えはダッシュボードのプロバイダー設定を変更して保存するだけです。

---

#### ローカルLLM共通の注意事項

- 送信コンテンツは **4,000 文字に自動制限**されます（小型モデルのコンテキスト上限への配慮）
- 応答が遅い場合はタイムアウトを長めに設定してください（目安: 90〜120秒）
- 4B〜8B の小型モデルでも動作しますが、要約品質はモデルによって異なります

### 9. タグがObsidianのデイリーノートにも書き込まれる 🏷️

タグ付き要約モードを使っている場合、AIが付けたタグは**ダッシュボードの履歴表示だけでなく、Obsidianのデイリーノートにも直接書き込まれます**。

書き込み形式:

```
- 09:25 [記事タイトル](https://example.com)
    - #IT・プログラミング #インフラ・ネットワーク ここにAI要約が入ります。
```

Obsidian のタグ検索（`#IT・プログラミング` など）で過去の記録を絞り込めます。

---

### 10. 「記録できなかったページ」を AI 要約なしで記録できる

キャッシュ制御やプライバシー保護でスキップされたページを、**AI要約なしで即座に記録**できるボタンが追加されました。「タイトルと URL だけ Obsidian に残したい」という場面で活用できます。

---

### 11. GDPR/CCPA 対応: データ削除・同意撤回 🗑️

ダッシュボードの「データ管理」セクションから：

- **全データ削除**: Chrome ローカルストレージに保存されたすべての設定・履歴を削除
- **プライバシー同意撤回**: データ収集への同意を撤回し、記録を停止

---

## 設定の引き継ぎについて

v5.0 へのアップデート後、**既存の設定はそのまま引き継がれます**。権限モデルの変更により、新しくアクセスするサイトでは初回に許可ダイアログが表示されますが、Tranco Top 1000 に含まれる主要サイトは引き続き記録されます。
