Yasumaro は、Chrome でタブを閉じるときに閲覧内容を AI で要約して Obsidian の日次ノートに自動記録する Chrome 拡張機能です。

この記事では v5.2 の機能を中心に、導入から日常的な使い方までを説明します。

---

## できること

- タブを閉じると自動的にページ内容を取得・AI 要約して Obsidian に保存
- OpenAI（GPT-4o など）・Google Gemini・Anthropic Claude・Groq、またはローカル AI（Ollama / LM Studio）を選んで利用可能
- 記録したくないドメインはフィルタリスト（uBlock Origin 形式も対応）で除外
- 個人情報（メールアドレス・電話番号・クレジットカード番号など）をマスクしてから AI に送信
- 日次ノートのどのセクションに書き込むかを指定可能
- カスタムプロンプトでAI要約のスタイルを自分好みに変更

---

## セットアップ

### 1. Obsidian Local REST API プラグインのインストール

Obsidian と連携するために [Local REST API](https://obsidian-local-rest-api.com) プラグインが必要です。

1. Obsidian の設定 → コミュニティプラグイン → 参照
2. 「Local REST API」で検索してインストール・有効化
3. プラグイン設定を開き、API キーをコピーしておく

### 2. 拡張機能のインストール

Chrome ウェブストアから「Yasumaro」をインストールします。インストール後、ツールバーのアイコンをクリックするとポップアップが開きます。

### 3. 初回設定

拡張機能アイコンをクリックして設定画面を開きます。

**Obsidian 接続設定:**

| 項目 | 設定値 |
|------|--------|
| API URL | `https://127.0.0.1:27124`（デフォルト） |
| API キー | Obsidian の Local REST API から取得したキー |
| 日次ノートのパス | Vault 内の日次ノートが保存されるフォルダ（例: `Daily Notes`） |

**AI プロバイダの選択:**

設定ダッシュボード（拡張機能アイコン → メニューボタン → 設定）で AI プロバイダを選択します。

- **OpenAI**: API キーを入力（GPT-4o / GPT-4o mini など）
- **Gemini**: Google AI Studio の API キーを入力
- **Claude**: Anthropic のAPIキーを入力
- **Groq**: Groq Cloud の API キーを入力
- **ローカル AI（Ollama）**: モデル名を入力（API キー不要）
- **ローカル AI（LM Studio）**: エンドポイントとモデル名を入力

## 基本的な使い方

### 自動記録

設定が完了したら、Web ページを閲覧してタブを閉じるだけです。拡張機能がバックグラウンドで次の処理を行います。

1. ページのテキストコンテンツを抽出
2. PII（個人情報）をマスク（設定で有効の場合）
3. AI に要約を依頼
4. Obsidian の今日の日次ノートに書き込み

### 手動記録

タブを閉じずに現在のページを記録したい場合は、ポップアップの「今すぐ記録」ボタンをクリックします。

### 記録の確認

ポップアップの「履歴」タブで記録済みのエントリを確認できます。AI 要約の内容・クレンジング状態・タグを閲覧できます。

---

## 記録をカスタマイズする

### ドメインフィルター

記録したくないサイトはドメインフィルターで除外します。設定ダッシュボードの「フィルター」パネルでドメインを追加できます。

uBlock Origin のフィルタリストをインポートすることもできます（「フィルターリストのインポート」ボタン）。

### カスタムプロンプト

AI へのプロンプトをカスタマイズできます。設定ダッシュボードの「プロンプト」パネルで編集します。

プリセットとして「要約」「技術的詳細」「アクションアイテム」「学習メモ」「読書ノート」の 5 種類があり、選んで使い始めることもできます。

### PII マスキング

設定ダッシュボードの「プライバシー」パネルで、AI に送信する前に個人情報をマスクするかどうかを設定できます。

マスク対象: メールアドレス・電話番号・クレジットカード番号・郵便番号・IPアドレスなど

## v5.2 で改善されたこと

v5.2 ではユーザーが直接体感できる変更はほとんどありませんが、内部的な信頼性が大幅に向上しています。

**Service Worker の安定性向上**: Chrome は拡張機能のバックグラウンドプロセス（Service Worker）をアイドル時に自動終了します。v5.2 では再起動後も処理状態が復元されるようになり、記録が途中で止まりにくくなりました。

**ローカル AI のセキュリティ**: Ollama や LM Studio を使っている場合、悪意ある Web コンテンツがプロンプトを書き換えようとする攻撃（Prompt Injection）への対策を追加しました。

## よくある質問

**Q. 記録が Obsidian に届かない**

1. Obsidian が起動しているか確認する
2. Local REST API プラグインが有効化されているか確認する
3. 拡張機能の接続テスト（設定ダッシュボード → テスト）を実行する
4. `https://127.0.0.1:27124` にブラウザでアクセスして証明書の信頼を許可する

**Q. AI 要約が空になる**

1. AI プロバイダの API キーが正しいか確認する
2. 設定ダッシュボードの「AI テスト」ボタンで接続確認する
3. ローカル AI の場合、Ollama / LM Studio が起動しているか確認する

**Q. 特定のサイトだけ記録したい**

ドメインフィルターで「すべてをブロック」し、記録したいドメインだけを許可リストに追加する方法は現在サポートしていません。除外したいドメインをフィルターに追加する形で運用してください。
