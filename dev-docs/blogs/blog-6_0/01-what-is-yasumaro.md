ブラウザで調べものをしていると、「あの記事、どこだったっけ」となることがよくありませんか。

私は毎日のように開発ドキュメントや技術ブログ、論文を読み漁っています。ブックマークは増える一方で、どのページで何を読んだのか、記憶があいまいになってしまうことが少なくありません。そんな体験から生まれたのが、Yasumaro という Chrome 拡張機能です。

## Yasumaro がやることはシンプルだ

あなたがウェブページを読んでいると、背後で Yasumaro が動きます。

ページに十分な時間滞在し、ある程度スクロールすると、そのページの内容を AI が要約します。要約結果はローカルの SQLite データベースに保存され、あとから検索して読み返すことができます。Obsidian を使っている場合は、デイリーノートにも自動追記されます。

必要なものは、Chrome 拡張機能と AI の API キーだけです。

**Obsidian は必須ではありません。**

この一点が、これまでの類似ツールとは大きく異なるところです。

## Obsidian なしで何ができるか

Yasumaro は「Obsidian がなくても完結する」設計になっています。拡張機能のダッシュボードから、以下のことがすべて行えます。

| 機能 | Obsidian なしでも利用可 |
|------|------------------------|
| 自動記録（AI要約付き） | ✅ |
| 手動記録 | ✅ |
| 履歴の一覧表示・検索 | ✅ |
| 全文検索（FTS5） | ✅ |
| カレンダー・タイムライン表示 | ✅ |
| エクスポート（Markdown / CSV / JSON） | ✅ |
| データの手動削除・保持期間設定 | ✅ |
| Obsidian への自動追記 | Obsidian 起動時のみ ✅ |

Obsidian 連携はあくまで「オプション」の位置づけです。設定画面で Obsidian 連携を OFF にしても、AI 要約とローカル保存はそのまま動き続けます。

## データはどこに保存されるか

保存先は、ブラウザ内の **OPFS（Origin Private File System）** 上に置かれた SQLite データベースです。ページをリロードしても、Chrome を終了しても、データは消えません。

```
[Chrome Extension]
   └── [Offscreen Document]
        └── [Web Worker]
             └── SQLite (OPFS) ← yasumaro.db
```

Service Worker からは直接 SQLite に触れないため、Offscreen Document 内の Web Worker を介して読み書きします。Manifest V3 の制約を満たしつつ、永続化を実現しています。

容量についても、無制限に近い形で運用可能です。必要に応じて「90日以上前のデータを削除」「10,000件を超えたら古いものから削除」といった保持ポリシーも設定できます。

## AI 要約の仕組み

Yasumaro は複数の AI プロバイダーに対応しています。

- **Groq**（無料枠あり・推奨）
- **Google Gemini**（無料枠あり）
- **OpenAI** / **Anthropic**
- **Ollama** などのローカル LLM（API キー不要）

ローカル LLM を選べば、要約処理も含めて完全にローカル完結できます。クラウド AI を使う場合でも、送信前に個人情報をマスクする「PII サニタイズ」機能が備わっており、メールアドレスや電話番号、クレジットカード番号などは自動的に伏せ字に変換されます。

## プライバシー設計

データが外部に流出しないよう、以下の設計を徹底しています。

- 閲覧履歴のデータはすべてブラウザ内に保存
- API キーは PBKDF2 + AES-GCM で暗号化して保管
- 初回起動時にプライバシー同意を取得（3回拒否で制限モードへ）
- プライベートページ（銀行・メールなど）は自動検出し、保存前に確認ダイアログを表示

「記録したくないサイト」はドメインフィルターで柔軟に制御できます。uBlock Origin 形式のフィルターリストをインポートすることも可能です。

## 今すぐ試すには

Yasumaro は Chrome Web Store からインストールできます。インストール後、ダッシュボードで AI プロバイダーを選んで API キーを入力すれば、あとは自動的に記録が始まります。

Obsidian をお持ちの方は、Local REST API プラグインの設定も追加すれば、並行してデイリーノートにも流し込まれます。

Obsidian を使っていない方は、拡張機能のダッシュボードから「SQLite History」タブを開いてみてください。これまでの閲覧履歴が、要約付きで一覧表示されています。検索バーにキーワードを入れると、日本語の部分一致検索で即座に絞り込めます。

## まとめ

Yasumaro は「ブラウザの記憶を、AI とともに残す」ツールです。Obsidian があると便利ですが、なくても十分に使えます。

あなたが今日読んだ記事の要約が、明日には検索一発で見つかる。それが Yasumaro の目指す体験です。
