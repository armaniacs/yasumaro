# Obsidian Weave v5.1.0 リリースノート

## 概要

v5.1.0では、ウェブページ要約の品質向上と、クレンジング統計の可視化を追加しました。

## 変更点

### 1. クレンジング削減率の可視化

Obsidian Weave がウェブページを記録するとき、バックグラウンドで4段階の「テキスト絞り込み」が走っています。

```
DOM全体 → コンテンツ候補 → Content Cleansing後 → AI Summary Cleansing後
pageBytes  candidateBytes    cleansedBytes          aiSummaryCleansedBytes
```

この数値は `chrome.storage.local` に蓄積され続けていましたが、v5.1ではこの4段階のバイト数をグラフと統計サマリーとして可視化します。

#### なぜ「削減率の見える化」が重要なのか

**トークンコストは入力サイズに比例する**

現在の主要AIプロバイダー（OpenAI、Anthropic、Google）はいずれもトークン単位の課金モデルを採用しています。要約精度とは独立して、入力テキストが長ければ長いほどコストは上がります。

典型的なニュースサイトのHTMLは300〜800 KB程度あります。そのうち本文として意味のある部分は多くても数十 KBです。差分のほとんどはナビゲーション、广告スクリプト、フッター、関連記事リストです。

削減率を可視化することで、「このサイトはほとんどノイズだ」「このサイトはクレンジングが効いていない」という判断ができるようになります。

**Lost in the Middle 問題**

スタンフォードの研究チームが2023年に発表した「Lost in the Middle」論文は、LLMに長い文書を与えたとき、文書の中間部分への注意が低下することを示しています。

| 答えの位置 | 正確率 |
|-----------|--------|
| 先頭 | 71% |
| 中間 | 52% |
| 末尾 | 68% |

入力を短くすることで、このリスクを下げられます。

#### 可視化の3要素

**統計サマリーカード**

AI Summary Cleansingパネルに3枚のカードを追加します。

| カード | 内容 |
|--------|------|
| 平均削減率 | `(pageBytes - 最終バイト) / pageBytes × 100` の平均値（%） |
| 累計削減量 | 全エントリの `pageBytes - 最終バイト` の合計（KB換算） |
| 集計対象 | 有効データのあるエントリ件数 |

**ファネルチャート（Canvas API）**

4段階の平均バイト数を横向きのファネルチャートとして描画します。Chrome ExtensionのCSP制約により外部チャートライブラリは使えないため、Canvas 2D APIで直接描画します。

```
pageBytes      ████████████████████████████  (100%)
candidateBytes ████████████████              ( 60%)
cleansedBytes  ████████████                  ( 45%)
aiCleansed     ████████                      ( 30%)
```

**プログレスバー（インライン表示）**

履歴リストの各エントリにプログレスバーを追加します。

```
コンテンツ抽出 — バイト: 248,320 → 12,840（94.8%削減）
[████████████████████████████████░░░░░░░░] 94.8%
```

#### パネル切り替え時の再描画修正

Dashboardのパネルナビゲーションで「AI Summary Cleansing」パネルに切り替えたとき、Canvasが非表示時に幅0で描画されるというバグを修正しました。

`requestAnimationFrame`を使用してパネルが表示された後に再描画を遅延させ、正しい幅計算を保証します。

### 2. AI要約の品質改善

v5.1では、ウェブページ要約の品質を改善する4つの施策を実装しました。

| 優先度 | 施策 | 変更規模 | 主な効果 |
|--------|------|----------|----------|
| 1 | リンク密度フィルタ デフォルトON | 定数2箇所 | ナビゲーションブロック除去 |
| 2 | デフォルトプロンプトに事実制約追加 | 文字列定数変更 | ハルシネーション抑制 |
| 3 | MMR相当の入力冗長除去 | 新規ファイル追加 | 重複トークン削減 |
| 4 | 文末正規化ポストプロセス | 新規ファイル追加 | 日本語情報密度向上 |

#### 施策1: リンク密度フィルタのデフォルトON化

ニュースサイトや技術ブログには、記事本文の下に「関連記事」「おすすめ記事」のリストが大量に並びます。これらは本文と同じDOM構造に存在するため、コンテンツ候補として取り込まれてしまい、要約に「関連記事として〜〜があります」のような内容が混入します。

AI Summary Cleansingの「リンク密度フィルタ」は、テキスト全体に占めるアンカーテキストの割合が70%以上かつ100文字以上の要素を除去します。この機能は既に実装・テスト済みでしたが、デフォルト値が `false` でした。v5.1でデフォルトを `true` に変更しました。

#### 施策2: デフォルトプロンプトに事実制約を追加

LLMのハルシネーションには2種類あります：

- **内在的ハルシネーション**: 入力文書の内容と矛盾する内容を生成する
- **外在的ハルシネーション**: 入力文書に存在しない情報を、事前学習知識から補完して生成する

後者は「一見もっともらしく見える」という厄介な性質があります。

デフォルトのシステムプロンプトに制約文を追加しました：

```
You are a helpful assistant that summarizes web pages effectively and concisely in Japanese.
Only use information explicitly stated in the provided content. Do not add facts, context, or details not present in the source text.
```

ユーザーがカスタムプロンプトを使用ている場合はこの変更の影響を受けません。

#### 施策3: MMR相当の入力冗長除去

**MMR（Maximal Marginal Relevance）** は1998年に提案された文書選択アルゴリズムです。コアアイデアは「関連性を最大化しつつ、すでに選択した内容との重複を最小化する」です。

v5.1では**文レベルの重複除去**を実装しました。Jaccard類似度を使って類似した文を重複として除去します。

```typescript
function jaccardSimilarity(a: Set<string>, b: Set<string>): number {
  let intersection = 0;
  for (const word of a) {
    if (b.has(word)) intersection++;
  }
  const union = a.size + b.size - intersection;
  return intersection / union;
}
```

**日本語bigram類似度**: 日本語テキストにbigramベースの類似度計算を追加しました。日本語文字が検出された場合、文字bigramを単語セットに含めることで、より正確なJaccard類似度計算が可能になりました。

```typescript
if (containsJapanese(cleaned)) {
  words.push(...getBigrams(cleaned));
}
```

**文区切りの保持**: 重複除去時に文の区切り文字が失われる問題を修正しました。文のテキストと区切り文字を個別に追跡する構造で保持します。

**threshold=0特別処理**: thresholdが0に設定された場合の明示的なパススルーを追加しました。

#### 施策4: 文末正規化ポストプロセス

日本語のAI要約では「〜しています」「〜することができます」という丁寧体（ます体）がよく使われます。これらは情報を持たず、常体（だ体）に変換することで文字数を減らし、情報密度を上げられます。

**安全な5パターンのみ変換**:

| 変換前 | 変換後 |
|--------|--------|
| `です。` | `だ。` |
| `でした。` | `だった。` |
| `ています。` | `ている。` |
| `ていました。` | `っていた。` |
| `でしょう。` | `だろう。` |

誤変換リスクを防ぐため、`ます。→ る。` や `ました。→ た。` は除外しました。

#### UI: Contentパネルへの設定追加

施策3と4はユーザーがOFFにできます。DashboardのContentパネルに設定を追加しました：

- **入力重複除去**: チェックボックスで有効/無効を切り替え
- **類似度閾値**: スライダーで0.5〜0.95の範囲で調整（デフォルト0.7）
- **文末正規化（日本語）**: チェックボックスで有効/無効を切り替え
