# Obsidian Weave v4.1 — AIが読んで、タグをつけてくれるようになりました

ダッシュボードの記録履歴を眺めていて、こう思ったことがあります。

「URLとAI要約が並んでいるだけだと、あとから探しにくい。」

obsidianを見れば、週に数十件の記録が積み上がっています。ただ、「あのページ、技術記事だったか、ニュースだったか」を思い出すのに要約を全部読まないといけません。タイトルで検索しても、うろ覚えのキーワードでは引っかかりません。

v4.1はその問題を解決します。AIが要約を生成するついでに、**カテゴリタグを自動で付与する**ようになりました。

**スクリーンショット説明**: Obsidian日次ノートに記録されたエントリ。タイトル、URL、AI要約テキストに加えて、カテゴリタグバッジ（例: 「IT・プログラミング」「ニュース」）が表示されている。

こんな具合で、カテゴリタグが自動でつきます。

---

## どんなふうに動くのか

<video src="promo.mp4" controls width="100%"></video>

記録が保存されるとき、AIが要約とあわせてタグを返します。「IT・プログラミング」「インフラ・ネットワーク」「ビジネス・経営」といったカテゴリの中から、ページの内容に合うものを選んで付けます。ユーザーが何かする必要はありません。

タグが付いたら、履歴パネルのバッジとして表示されます。バッジをクリックすれば、同じカテゴリの記録だけを絞り込めます。タグパネルに切り替えてカテゴリ名を選んでも同じ操作ができます。

手動でタグを編集することもできます。「自動では違うカテゴリが付いた」「追加したい」という場合は、エントリの編集ボタンから変えられます。

---

## デフォルトのカテゴリは10種類

最初から用意されているカテゴリはこちらです。

| カテゴリ | こんなページに付きます |
|---------|-----------------|
| IT・プログラミング | 技術ブログ、GitHubのドキュメント |
| インフラ・ネットワーク | クラウド、ネットワーク構成の記事 |
| サイエンス・アカデミック | 論文、研究解説 |
| ビジネス・経営 | 経営戦略、マーケティング |
| ファイナンス・投資 | 株、経済ニュース |
| ライフスタイル・健康 | 生活改善、フィットネス |
| エンタメ・カルチャー | 映画、音楽、ゲーム |
| ニュース・時事 | 時事記事、速報 |
| 学習・教育 | 勉強法、教育コンテンツ |
| その他 | 上記に当てはまらないもの |

デフォルトカテゴリは編集も削除もできません。かわりに、ダッシュボードの「タグ設定」からユーザー独自のカテゴリを追加できます。「法律・契約」「レシピ」「競合調査」など、自分の閲覧パターンに合わせて増やせます。

## タグ付きモードをオンにするには

デフォルトでは、タグ付き要約モードはオフになっています。オンにするにはダッシュボード → タグ設定で「タグ付き要約モード」のトグルを有効にします。

この設計にした理由があります。タグを生成するには、AIプロンプトに「カテゴリの一覧から選んで返せ」という指示を追加する必要があります。これはトークン数を増やします。AIのAPI利用料を気にしているユーザーに黙ってオンにするのは適切でないと判断しました。

オンにすると、AIの返答フォーマットが変わります。従来は「要約テキスト」だけだったものが、「要約テキスト + タグ情報（JSON）」の形式になります。バックエンドがそのJSONを解析して `tags` フィールドに保存する仕組みです。

## ダッシュボードも整理しました

タグ機能の追加と合わせて、ダッシュボードの見た目を全体的に整えました。

ステータスパネルの絵文字アイコン（✓ ✗ ⚠）をSVGアイコンに置き換えました。絵文字はOSによってレンダリングが違い、サイズや色が安定しません。SVGに変えたことでどの環境でも同じ表示になります。

サイドバーの各メニュー項目にアイコンを追加しました。テキストだけより、視覚的に目的の設定画面に飛びやすくなりました。

パネル切り替えのアニメーションも手を入れました。前のバージョンでは CSS 変数の定義漏れでパネルが真っ暗になるバグがありました。アニメーションを廃止してシンプルな `display` 切り替えに変更しています。

## 細かいバグも直しました

小さいですが、長く残っていたものをいくつか修正しました。

**診断パネルのタイトルが赤い大文字になる問題。** 未定義のi18nキーを `data-i18n` 属性として指定していたため、i18nシステムがキー名をそのままスタイル付きで表示していました。`data-i18n` を削除して直接テキストを書くように変更しました。

**ドメインフィルターパネルの初回表示が空になる問題。** パネルを初めて選んだとき、ドメインリストが空で表示されることがありました。`loadDomainSettings()` の完了を待たずに描画が走っていたのが原因で、`async/await` を使って順序を正しました。

**タグフィルターインジケーターのXSSリスク。** ユーザーが定義したカテゴリ名を `innerHTML` に直接書き込んでいました。`textContent` ベースの操作に変更しています。

**タグ編集モーダルのフォーカス管理。** モーダルを開いたあとTabキーを押すと、フォーカスがモーダルの外に出てしまっていました。`focusTrapManager` を適用し、WCAG 2.1 Level AA の要件に合わせました。

## タグを使った記録の探し方

タグが付くと、大きく2通りの使い方ができます。

**バッジからフィルター。** 履歴エントリのタグバッジをクリックすると、同じカテゴリのエントリだけが残ります。「IT・プログラミングの記事をまとめて見たい」という場面で使います。

**タグパネルから移動。** ダッシュボードのサイドバーから「タグ」を選ぶと、カテゴリごとの件数が確認できます。カテゴリをクリックすると履歴パネルに切り替わり、そのタグで絞り込まれた状態で表示されます。

どちらも同じ結果になります。操作の起点が違うだけです。

## インストール・更新

```bash
git pull origin main
npm run build
```

ビルド後、`chrome://extensions` で「更新」ボタンをクリックしてください。

タグ付き要約モードを有効にするにはダッシュボード → タグ設定 → トグルをオン。すでに保存済みの記録にはタグが遡及して付くことはありません。今後記録されるページから適用されます。
