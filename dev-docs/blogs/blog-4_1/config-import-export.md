設定画面を開くたびに「また一からやり直し…」と思ったことはないでしょうか。

新しいPCへの移行。ブラウザのプロファイルを分けて使っているとき。あるいは、ちょっと設定を壊してしまって元に戻したいとき。Obsidian Weave 4.1 では、**設定をまるごとエクスポート・インポートできる機能**を追加しました。

## 何が変わったか

Obsidian Weave の設定は、Chrome の内部ストレージ（`chrome.storage.local`）に保存されています。以前はこの中身を直接触る方法がなく、引越しのたびに手動で再設定するしかありませんでした。

今回追加したエクスポート/インポート機能を使うと、以下の設定をすべて JSON ファイルに書き出せます。

| 設定カテゴリ | 内容 |
|------------|------|
| Obsidian 接続先 | プロトコル・ポート・保存パス |
| AI プロバイダー | プロバイダー種別・モデル名・Base URL |
| 記録フィルター | ドメインホワイトリスト / ブラックリスト |
| プライバシー設定 | PII マスク・ログ保護の各フラグ |
| タグ設定 | タグ付き要約モード・カスタムカテゴリ |
| フォーマット設定 | uBlock ルール・シンプルフォーマット |

**APIキーは意図的にエクスポートから除外しています。** 設定ファイルが誰かの手に渡っても、キーは漏れません。インポート後も、既存のAPIキーはそのまま保持されます。

## エクスポートの手順

1. ダッシュボードを開きます（拡張機能アイコンをクリック → 右上の「ダッシュボード」アイコン）
2. サイドバーから **「設定」** パネルを開きます
3. 下部の **「設定をエクスポート」** ボタンをクリックします
4. `obsidian-weave-settings-YYYYMMDD-HHmmss.json` というファイルがダウンロードされます

ファイル名にはエクスポート日時が自動で入るので、バックアップが複数あっても区別できます。

## インポートの手順

1. 同じく **「設定」** パネルを開きます
2. **「設定をインポート」** ボタンをクリックします
3. エクスポートしたJSONファイルを選択します
4. 設定が即座に反映されます

インポート後はページのリロードは不要です。APIキーの再設定だけ忘れずに行ってください。

## セキュリティの仕組み

エクスポートファイルには **HMAC 署名** が付きます。インポート時にこの署名を照合することで、ファイルが改ざんされていないか確認しています。

```
エクスポート時:
  設定データ → HMAC署名を付加 → JSON ファイル

インポート時:
  JSON ファイル → 署名を照合 → 設定を適用
```

署名の照合に失敗した場合（拡張機能の再インストール後など）は、確認ダイアログが表示されます。自分でエクスポートしたファイルであれば「OK」で強制インポートできます。

## Q. APIキーはどうすればいい？

Q. 新しい環境でインポートしたら、APIキーはどこに行った？

A. 大丈夫です。APIキーはエクスポートに含まれません。インポート後、設定パネルの各プロバイダー欄にAPIキーを入力すれば完了です。設定ファイルを共有・公開しても、キーが流出する心配はありません。

## まとめ

- エクスポートで設定を JSON に書き出せます
- インポートで別の環境や同じ環境に即座に適用できます
- APIキーは除外されるので、ファイルを安全に保管・共有できます
- HMAC署名で改ざん検知に対応しています

PC の買い替えや、設定を誤って壊してしまったときの備えとして、定期的なエクスポートをおすすめします。
