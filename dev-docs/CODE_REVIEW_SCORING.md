# Code Review Scoring Criteria

このドキュメントは、Obsidian Weaveプロジェクトで使用しているChecking Teamによるコードレビューのスコアリング基準を定義します。

## スコアリングシステム

| スコア | 範囲 | 基準 | アクション |
|:-----:|-----:|------|---------|
| **S** | 90-100 | 優秀な品質、改善点なし | リリース準備完了 |
| **A** | 80-89 | 高品質、軽微な改善提案あり | 改善推奨、但しリリース可能 |
| **B** | 70-79 | 良好品質、いくつかの改善推奨 | 改善推奨、リリース前に対応 |
| **C** | 60-69 | 改善必要、技術的負債あり | 改善必須、リリース前に対応 |
| **D** | 50-59 | 懸念あり、早急な対応推奨 | 改善必須、リリース延期推奨 |
| **E** | 0-49 | 重大な問題、修正必須 | リリース禁止、修正必須 |

## 各エキスパートの評価領域

### Red Team Leader（セキュリティ：攻撃的観点）
**評価視点:**
- 注入脆弱性（XSS、SQLインジェクション等）
- 認証・認可の抜け穴
- 機密情報の漏洩リスク
- タイミング攻撃などの暗号学的脆弱性

**スコア調整項目:**
- 重大な脆弱性: -30点
- 中程度の脆弱性: -15点
- 軽微な脆弱性: -5点

### Blue Team Leader（セキュリティ：防御的観点）
**評価視点:**
- 入力検証とサニタイゼーション
- 安全なデフォルト設定
- 秘密管理の適切性
- セキュアなコーディングプラクティス

**スコア調整項目:**
- 防御策の欠落: -20点
- 非安全なデフォルト: -10点
- 推奨されるセキュアプラクティスの違反: -5点

### System Architect（システムアーキテクチャ）
**評価視点:**
- 全体構造の整合性
- コンポーネント間の境界と依存関係
- 設計に一貫性
- スケーラビリティと拡張性

**スコア調整項目:**
- アーキテクチャ違反: -20点
- 不適切な責務分離: -10点
- 一貫性の欠如: -5点

### UI Expert（ユーザー体験）
**評価視点:**
- UIの直感性
- ユーザーフローの最適性
- フィードバックの適切性
- ユーザー体験の全体的な質

**スコア調整項目:**
- UXの重大な問題: -15点
- ユーザー体験の未熟さ: -8点
- 軽微なUX改善点: -3点

### Tuning Expert（パフォーマンス）
**評価視点:**
- 実行速度と効率性
- メモリ使用量
- アルゴリズムの計算量
- ユーザー応答性

**スコア調整項目:**
- 重大なパフォーマンス問題: -20点
- 妥当化の余地のあるパフォーマンス改善: -10点
- 軽微な最適化機会: -5点

### SRE/Ops Specialist（信頼性・運用）
**評価視点:**
- エラーハンドリングと回復
- ログとメトリクス
- デプロイ耐性
- 観測容易性

**スコア調整項目:**
- 運用上の重大な問題: -25点
- ロギング不足: -10点
- エラー処理不備: -8点

### Maintainability Guardian（保守性）
**評価視点:**
- 凝集度と結合度
- DRY原則の遵守
- コードの可読性
- デッドコードの有無

**スコア調整項目:**
- 保守性の重大な問題: -20点
- コード重複やDRY違反: -10点
- 読みやすさの改善点: -3点

### Domain Logic Expert（ビジネスロジック）
**評価視点:**
- 仕様通りに実装されているか
- ビジネス要件の正確性
- エッジケースの扱い
- ドメイン知識の適切性

**スコア調整項目:**
- 仕様との不一致: -25点
- エッジケースの欠落: -15点
- 仕様の未定義箇所: -5点

### i18n Expert（国際化）
**評価視点:**
- ロケール対応の完全性
- 日付・時刻のタイムゾーンの扱い
- 多言語拡張性
- i18n APIの正しい使用

**スコア調整項目:**
- i18n違反: -15点
- ロケール未対応のユーザー向け文字列: -8点
- 拡張性の改善点: -3点

### Accessibility Advocate（アクセシビリティ）
**評価視点:**
- WCAG準拠
- スクリーンリーダー対応
- キーボードナビゲーション
- 色コントラスト

**スコア調整項目:**
- 重大なアクセシビリティ違反: -20点
- WCAG未準拠: -10点
- 改善可能なアクセシビリティ: -5点

### Compliance & Privacy Guard（コンプライアンス・プライバシー）
**評価視点:**
- 法的規制の遵守（GDPR、CCPA等）
- プライバシー保護の適切性
- ライセンスの遵守
- データ保護の適切性

**スコア調整項目:**
- 重大なコンプライアンス違反: -30点
- プライバシー保護の不足: -20点
- 文書化不足: -5点

### Documentation Architect（ドキュメント）
**評価視点:**
- READMEとAPI定義の整合性
- コメントの質と量
- 開発者体験 (DX)
- ドキュメントの一貫性

**スコア調整項目:**
- ドキュメント重大な不備: -15点
- API定義の不一致: -10点
- 軽微な改善点: -3点

### Data Integrity Expert（データ整合性）
**評価視点:**
- データスキーマの妥当性
- 整合性制約の適切性
- マイグレーション時のリスク管理
- データの信頼性

**スコア調整項目:**
- 整合性違反: -25点
- マイグレーションリスク: -15点
- スキーマの改善点: -5点

### Test Experts（テスト）
**評価視点:**
- テストカバレッジ
- テストの品質と信頼性
- 不可解なテスト失敗の有無
- 指摘の検証可能性

**スコア調整項目:**
- テスト重大な欠損: -25点
- カバレッジ不足: -15点
- 軽微な改善点: -5点

## 優先度の基準

各指摘事項には優先度が割り当てられます：

| 優先度 | 定義 | 対応 |
|:-----:|------|------|
| **Critical** | セキュリティ、クリティカルなバグ | 即時対応必須 |
| **High** | 重要な機能、ユーザー体験 | 次のリリースまでに対応 |
| **Medium** | 設計・パフォーマンス | 時期を決めて対応 |
| **Low** | 軽微な改善、ドキュメント | 随時対応 |

## 総合スコアの算出

1. 各エキスパートのスコア（100点満点）を算出
2. エキスパートごとに重み付け均等として、平均スコアを算出
3. 指摘事項の優先度と重要度を考慮して、最終スコアを算出

## レビュー結果のアクション

### S/A (80-100点)
- リリース準備完了
- 軽微な改善提案のみ

### B (70-79点)
- 改善推奨
- PR前または次回リリース前に対応

### C (60-69点)
- 改善必須
- リリース前に対応

### D (0-59点)
- 改善必須
- リリース延期推奨または修正必須
