# Code Review Scoring Criteria

このドキュメントは、Yasumaroプロジェクトで使用している Checking Team（`~/.claude/skills/checking-team`）によるコードレビューのスコアリング基準を定義します。

> スキル本体の仕様が正で、このドキュメントはその要約です。詳細や更新の際は `references/agent-common.md`（共通スコア式・出力形式）と `references/workflow.md`（フェーズ・レポート形式）を参照してください。

## スコアリングシステム

| ランク | スコア範囲 | 基準 | アクション |
|:-----:|-----:|------|---------|
| **S** | 90-100 | 優秀な品質、改善点なし | リリース準備完了 |
| **A** | 80-89 | 高品質、軽微な改善提案あり | 改善推奨、但しリリース可能 |
| **B** | 70-79 | 良好品質、いくつかの改善推奨 | 改善推奨、リリース前に対応 |
| **C** | 60-69 | 改善必要、技術的負債あり | 改善必須、リリース前に対応 |
| **D** | 50-59 | 懸念あり、早急な対応推奨 | 改善必須、リリース延期推奨 |
| **E** | 0-49 | 重大な問題、修正必須 | リリース禁止、修正必須 |

## 共通スコア式

全22観点は、個別のスコア加減表ではなく **共通の1つの式** で評価されます（観点ごとに基準がバラつくと相対比較ができなくなるため）。

```
スコア = 100 - (High指摘数 × 20) - (Medium指摘数 × 5)
最小値: 0、最大値: 100
```

| 優先度 | 定義 | スコア影響 |
|:------:|-----|:---------:|
| **High** | 修正しないとリリース不可・本番障害・法的リスクのある問題 | -20 |
| **Medium** | 修正推奨だが即時リリースは可能な問題 | -5 |
| **Low** | 改善提案・Nice to have | 0（スコアに影響しない） |

総合スコアは、**実際に実行された観点**のスコアの単純平均です（加重はしない）。関連性マップによりスキップされた観点や、起動に失敗して結果が得られなかった観点は平均に含めません。

## 22の評価観点

### Wave 1: コアレビュアー（常に実行）

| 観点 | 評価視点 |
|------|---------|
| Red Team Leader | 攻撃的観点からの脆弱性特定（OWASP Top 10、開発者起点のリスク、論理的脆弱性） |
| Blue Team Leader | 防御的観点からの安全性検証（入力バリデーション、認証・認可、エラーハンドリング） |
| System Architect | 全体構造・技術選定の妥当性評価、専門家間のコンフリクト調整 |
| Maintainability Guardian | 長期的な保守性・可読性（設計レベルの正しさ、DRY原則） |
| Legacy Bridge Architect | 内部実装の変更による既存呼び出し元への破壊的影響の防止 |

### Wave 2: スペシャリスト（4クラスタ・16観点、関連性マップにより一部スキップされうる）

| クラスタ | 含まれる観点 |
|---------|------------|
| UX/Frontend | UI Expert, Accessibility Advocate, i18n Expert, Documentation Architect |
| Ops/Performance | Tuning Expert, SRE/Ops Specialist, FinOps Consultant, Edge & Mobile Strategist |
| Governance/Risk | Compliance & Privacy Guard, Ethics & Bias Auditor, Supply Chain & Dependency Sentinel, API & Contract Negotiator |
| Code Quality | Domain Logic Expert, Data Integrity Expert, Refactoring Evangelist, DX Advocate |

### Wave 3: テスト（Wave 1+2 で High/Medium 指摘が1件以上あった場合のみ実行）

| 観点 | 評価視点 |
|------|---------|
| Test Experts | Wave 1+2 の指摘に対するテストケース作成・可能な修正 |

各観点の詳細なチェックリストは `~/.claude/skills/checking-team/references/` 配下の個別ファイル（`red-team.md` 等）を参照してください。

## 優先度の基準

| 優先度 | 定義 |
|:-----:|------|
| **High** | 修正しないとリリース不可・本番障害・法的リスクのある問題 |
| **Medium** | 修正推奨だが即時リリースは可能な問題 |
| **Low** | 改善提案・Nice to have |

## レビュー結果のアクション

- **High または Medium 指摘が1件以上ある場合**: 自動修正フェーズに進む（破壊的変更を伴う指摘のみユーザーに確認）
- **Low のみ、またはゼロの場合**: レポートを提示して終了

### ランク別の目安

- **S/A (80-100点)**: リリース準備完了、軽微な改善提案のみ
- **B (70-79点)**: 改善推奨、次回リリース前に対応
- **C (60-69点)**: 改善必須、リリース前に対応
- **D/E (0-59点)**: 改善必須、リリース延期推奨または修正必須
