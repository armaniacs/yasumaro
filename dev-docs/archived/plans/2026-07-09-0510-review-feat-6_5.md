## 重要指摘事項（優先度順）

### [High] 生ページ本文(content)の SQLite 永続化がデータ最小化・PII 原則に違反（重複指摘: Red Team + Compliance & Privacy を統合）
- 指摘者: Red Team Leader, Compliance & Privacy Guard
- 場所: `src/offscreen/schema.ts:22`, `src/offscreen/opfsWorker.ts:32-55`, `src/background/pipeline/RecordingPipeline.ts:180`
- 影響: PBI-3 の `content` カラムにコンテンツスクリプトが抽出した「生」ページ本文がそのまま格納される。`content` は PII サニタイザを通らず、デフォルト保持期間 `CONTENT_RETENTION_DAYS: null`（無制限）。医療・金融・認証情報を含む全文がローカルに無期限保存され、GDPR/CCPA のデータ最小化に反する。
- 対処: (a) 保存前に `piiSanitizer` を通す、または (b) `CONTENT_RETENTION_DAYS` のデフォルトを `null`→14/30 日に変更、(c) 初回セットアップで content 保存の明示的同意 UI を追加。→ **設計判断が必要（ユーザー確認）**

### [High] ローカル Markdown 書き出しが録画ごとに `chrome.downloads.download` を発行
- 指摘者: System Architect
- 場所: `src/background/pipeline/steps/saveLocalMarkdownStep.ts:92-104`
- 影響: 全録画完了ごとにダウンロードUIが出現し UX 劣化。合計ファイルを毎回再ダウンロードするため書き込み量が O(n²)、dataURL サイズ上限(~2MB) に到達するリスク。
- 対処: Offscreen Document 経由の File System Access API または日次バッチ書き出しに変更。新規 `downloads` 権限の正当性説明も必要。→ **設計判断が必要（ユーザー確認）**

### [High] `tagClusterLoading.ts` に日本語ハードコード（i18n 機構のバイパス）
- 指摘者: i18n Expert
- 場所: `src/dashboard/tagClusterLoading.ts:20-24`
- 影響: 4段階ローディングラベルがソースに直接ハードコードされ、日本語以外は読めない。
- 対処: `getMessage()` 経由に変更し `_locales/{en,ja}/messages.json` にキー追加。**機械的修正可能（低リスク）**

### [High] ALTER TABLE マイグレーションが全エラーをサイレント握り潰し
- 指摘者: Data Integrity Expert
- 場所: `src/offscreen/sqlite.ts:297-314`, `src/offscreen/opfsWorker.ts:217-236`
- 影響: ディスクフル・DB破損も「duplicate column」と同じように隠蔽。
- 対処: catch で duplicate column のみ無視し、他は warn/再スロー。→ **Wave 3 で修正済み（resolved）**
