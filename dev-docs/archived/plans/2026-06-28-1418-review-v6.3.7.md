## 重要指摘事項

### [Medium] HTTP 保存時にユーザー確認が不要
- **指摘者**: Red Team
- **場所**: `src/popup/settings/fieldValidation.ts:78-93`
- **影響**: `validateProtocol` が HTTP 時に警告を表示するが `true` を返すため、警告を認識せずに平文送信設定を保存できる。`settingsSaver.ts` の `handleSaveAndTest` はバリデーション戻り値のみで保存可否を判断。
- **対処**: HTTP 選択時、保存前に `confirm()` または専用モーダルで明示的な同意を取得する。または「理解した」チェックボックスを必須にする。

### [Medium] `_validateProtocol` が配列入力を受理する可能性
- **指摘者**: Red Team
- **場所**: `src/background/obsidianClient.ts:148-165`
- **影響**: `String(["http"])` → `"http"` となる。ストレージに配列が書き込まれた場合、型アサーション `as string` をバイパスする。
- **対処**: `typeof protocol !== 'string'` のガードを追加。文字列以外は `'https'` にフォールバック。
