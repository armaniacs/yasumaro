## 重要指摘事項（優先度順）

### [High] 🎯 デュアルライト (chrome.storage + SQLite) の移行未完了と調停メカニズム欠如
- **指摘者:** System Architect, Data Integrity Expert
- **場所:** `src/background/pipeline/steps/saveSqliteStep.ts`, `saveMetadataStep.ts`, `src/utils/storage/settingsStore.ts`
- **影響:** 同一データが chrome.storage.local と SQLite (OPFS) の両方に書き込まれている。一方の書き込みが成功し他方が失敗した場合の調停処理が存在しない。`pendingSqliteQueue` は SQLite 失敗時のみの退避であり、chrome.storage 書き込み失敗時にはデータが消失する。Service Worker の予期せぬ終了で不完全なデュアルライト状態が残るリスクがある。
- **対処:** 移行完了の明確なマイルストーンを設定し完了させる。移行完了までは書き込みごとにトランザクション的保証を持つ調停ロジックを導入する。デフォルトの `LEGACY_DUAL_WRITE_ENABLED` を `false` に変更する。

### [High] 🎯 `service-worker.ts` が神ファイル (God Object, 654行)
- **指摘者:** Maintainability Guardian, Refactoring Evangelist
- **場所:** `src/background/service-worker.ts` (全654行)
- **影響:** 初期化・マイグレーション・メッセージルーティング・オフラインキュー処理・セッション管理・コンテキストメニュー登録・バッジ更新・プライバシーパイプライン —— 少なくとも5つ以上の責務が1ファイルに混在。新機能追加時の変更影響範囲が大きく、merge conflict のリスクが高い。
- **対処:** 初期化オーケストレーター、メッセージルーティング、オフラインキュー監視、ライフサイクル/タブイベントを独立モジュールに抽出。service-worker.ts はイベントリスナー登録のみの薄いエントリポイントにする。

### [High] 🎯 migrateToSingleSettingsObject の破壊的一方向マイグレーション
- **指摘者:** Legacy Bridge Architect
- **場所:** `src/utils/storage/settingsStore.ts:191-201`
- **影響:** 旧 per-key 形式のストレージを単一 `settings` オブジェクトに移行後、per-key 群を完全削除する。これにより拡張機能のダウングレード（旧バージョンへのロールバック）が不可能になり、`settings` オブジェクト破損時に全設定が復旧不能になる。
- **対処:** per-key 削除を即時削除から「設定変更後○日経過したら削除」の遅延削除に変更する。最低限、削除前に settings オブジェクトの内容をバックアップキーに保存する。

### [High] 🎯 `web_accessible_resources` が内部 JS バンドルを全オリジンに露出
- **指摘者:** Red Team Leader
- **場所:** `wxt.config.ts:67-82`
- **影響:** `chunks/*.js` や `assets/*.js` が全 Web サイトから読み取り可能。内部ロジック・メッセージ形式・API エンドポイントの解析、WASM バイナリの取得による SQLite 構造分析のリスクがある。
- **対処:** 実際に Web ページから動的読み込みが必要なもののみに `matches` を制限する。

### [High] 🎯 オフラインキューリトライが全パイプラインを再実行し追加 API コストを発生
- **指摘者:** FinOps Consultant, Tuning Expert
- **場所:** `src/background/service-worker.ts:436-459`
- **影響:** `recordingLogic.record()` が常に AI 要約を実行する設定の場合、Obsidian 書き込み失敗のリトライ時に本来不要な AI API コールが発生。最大3回の追加 API コール + 5分間隔のアラームリトライが発生しうる。
- **対処:** リトライハンドラで `job.type === 'obsidian_sync'` と判定した場合、Obsidian 書き込みのみを再試行し AI 要約をスキップする。

### [High] 🎯 public/PRIVACY.md が docs/PRIVACY.md の最新更新に未追従
- **指摘者:** Compliance & Privacy Guard
- **場所:** `public/PRIVACY.md` vs `docs/PRIVACY.md`
- **影響:** Chrome Web Store 版プライバシーポリシーに PII サニタイゼーションの詳細、OPFS SQLite 主体の保存、PBKDF2 反復回数 (600,000回) の説明が欠落。Store 審査で虚偽の説明とみなされるリスクがある。
- **対処:** `public/PRIVACY.md` を `docs/PRIVACY.md` の内容に同期する。
