# はじめてのYasumaro — Obsidianなしで今日から使う閲覧履歴ロガー

「AIで閲覧履歴を要約して記録してくれる拡張機能」と聞くと、真っ先に浮かぶ疑問は「で、Obsidianって入れてないとダメなの？」ではないでしょうか。

答えは「いいえ」です。Yasumaroは、Obsidianを一切セットアップしなくても、その日見たページの要約をMarkdownファイルとしてダウンロードフォルダに残してくれます。この記事では、インストールから初回の記録確認、日々の使い方までを順番に見ていきます。

## なぜObsidianなしでも動くのか

Yasumaroという名前は、『古事記』を編纂した太安万侶（おおのやすまろ）に由来します。太安万侶は、稗田阿礼が暗誦した膨大な記憶を文字に起こし、口承の言葉を永続的な「記録」へと変えた人物です。当時は漢字を独自のルールで組み合わせて日本語の響きを表現するという、現代のプログラミングに通じる編集能力を発揮しました。

この拡張機能もそれと同じで、ブラウザで目にしたページ（＝一時的な記憶）を自動で文字に起こし、あとから振り返れる記録として残します。だから記録先はObsidianである必要がなく、ローカルMarkdownファイルでもその役割を果たせるのです。

Obsidian連携を使う場合、Local REST APIプラグインのインストール・証明書の設定・APIキーのコピーといった手順が必要になります。これは決して難しくはないのですが、「まず使ってみたい」人にとっては小さくないハードルです。ローカルMarkdown書き出し機能は、この手順を丸ごとスキップして、ダッシュボードのチェックボックスひとつで記録を始められるようにしています。

Obsidianを使う/使わないは、あとからいつでも切り替えられます。両方を同時にONにすることもできるので、「とりあえずローカルに溜めて、気が向いたらObsidian Vaultに取り込む」という運用も可能です。

## インストールする

Chrome Web StoreからYasumaroをインストールします。ソースからビルドする場合は `npm install` → `npm run build` で `dist/chromium-mv3` を `chrome://extensions` の「パッケージ化されていない拡張機能を読み込む」から読み込みます。

インストール後、拡張機能アイコンをクリックするとポップアップが開きます。初回はプライバシー同意モーダルが表示されます。

## 最初にすること：プライバシー同意とAI設定

### プライバシー同意モーダル

Yasumaroは閲覧ページの内容をAIに送信して要約を作るため、初回起動時に同意モーダル（`role="dialog"` の `<dialog>` 要素）が表示されます。ここで拒否した場合、30日後に再度モーダルが表示される仕様になっています（GDPR第7条を意識した設計です）。同意すると、記録機能が有効になります。

### AIプロバイダを選ぶ

ポップアップ右上の「⚙」アイコンから設定画面（ダッシュボード）を開き、サイドバーの「初期設定」パネルでAIプロバイダーを選びます。選択肢は次の6種類です。

| プロバイダー | 用途 |
|------------|------|
| Google Gemini | クラウドAI。デフォルトのおすすめ |
| OpenAI Compatible | Groqなど、OpenAI互換API全般 |
| OpenAI Compatible 2 | ローカルLLMなど、2つ目の互換プロバイダー枠 |
| LM Studio | ローカルで動くLM Studio |
| Ollama | ローカルで動くOllama |
| OpenAI Compatible (Models.dev) | Models.devのモデル一覧から選択 |

クラウドに何も送りたくない場合は、LM StudioやOllamaを使ったローカルAI(`local_only`モード)を選べます。ネットワーク接続不要で完結するため、プライバシーを最優先したい人向けの構成です。

APIキーはPBKDF2 + AES-GCMで暗号化されて保存されるので、平文のままローカルに残ることはありません。

## ローカルMarkdown書き出しを有効にする

ここが「Obsidianなしで使う」の核心部分です。ダッシュボードの「初期設定」パネルにある「ローカル Markdown 書き出し」セクションで、チェックボックスをONにします。

```
[x] ローカル Markdown に書き出す
    閲覧履歴を Markdown ファイルとしてブラウザのダウンロードフォルダに
    保存します（例: ~/Downloads/Yasumaro/）。
```

ONにすると、書き出しタイミングを4つから選べます。迷ったら「アイドル時 / 30分ごと」を選んでおくのが無難です。記録のたびにダウンロード通知が出て煩わしい、ということがありません。

| モード | こんな人向け |
|-------|------------|
| 手動のみ | 自分のタイミングでまとめてエクスポートしたい |
| 即時 | すぐにファイルを確認したい（通知は多め） |
| アイドル時 / 30分ごと | 普段使いにおすすめ。気づいたら溜まっている |
| 日付が変わったとき | 1日1回、前日分をまとめて回収したい |

保存先フォルダ名も変更できます（デフォルトは `Yasumaro`）。

## 実際にページを記録してみる

適当なページを開いて、ポップアップの「📝 Record Now」ボタンを押します。ただし、記録には条件があります。

1. ページの滞在時間が設定した最小値を超えている
2. スクロール深度が設定した最小値を超えている
3. ドメインフィルターやプライバシー検出（ログイン画面や決済ページなど）を通過している

条件を満たさないページは記録されません。単に開いただけでは記録されない、というのがYasumaroの基本方針です。「即時」モードにしている場合は、記録の直後（最短1分間隔）にMarkdownファイルが更新されます。「アイドル時」モードなら、しばらく待つかブラウザがアイドルになったタイミングでまとめて反映されます。

## 書き出されたファイルを確認する

`~/Downloads/Yasumaro/` を開くと、`YYYY-MM-DD.md` という名前で日付ごとのファイルが増えていきます。中身はこんな形式です。

```markdown
# 2026-07-05

- 14:30 [ページタイトル](https://example.com)
    - AI が生成した要約テキスト
- 15:00 [別のページ](https://example.com/page2)
    - 2番目のページの要約
```

見出しが日付、箇条書きが訪問したページとその要約という、シンプルな構造です。Obsidianを使い始めたときにも、このファイルをそのままVaultに放り込めば違和感なく読み込めます。

## 既存の履歴をあとからMarkdown化する

すでにいくつか記録が溜まっている状態から書き出し機能をONにした場合、過去分は自動では出力されません。手動エクスポートを使います。

- ダッシュボードの「初期設定」→「ローカル Markdown 書き出し」で開始日・終了日を指定して「エクスポート」
- 「ログをエキスポート」パネルで日付範囲を指定してエクスポート
- 「履歴」パネルの「すべて Markdown に書き出す」ボタン（全期間対象）

3つとも内部的には同じ変換ロジックを使っているので、どこから実行しても結果は同じです。使いやすい導線を選べば十分です。

## 日常運用で見るところ

慣れてきたら、ダッシュボードの以下のパネルをたまに覗くと便利です。

- **SQLite History**: トークン数・処理時間・使用したAIプロバイダーまで含めた記録の一覧と全文検索
- **Tag Cluster**: 記録に付いたタグ同士の関連をグラフで可視化。クリックすると該当タグで履歴を絞り込める
- **診断**: SQLiteの状態確認、メトリクス補完、旧ストレージの削除など

## よくあるつまずき

**Q. ファイルがダウンロードされない**
「ローカル Markdown に書き出す」がONか、記録条件（滞在時間・スクロール深度）を満たしているかを確認してください。それでも解決しない場合は、Service Workerのコンソールで `[LocalMD]` プレフィックスのログを確認します（`chrome://extensions` → Yasumaroの「Service Worker」→「検証」）。

**Q. 毎回ダウンロード通知が出て気になる**
`chrome://settings/downloads` の「ダウンロードが完了したとき、ダウンロード一覧を表示する」をOFFにすると静かになります。

**Q. あとからObsidianも使いたくなった**
同じ初期設定パネルにObsidian連携の項目があります。Local REST APIプラグインを導入し、APIキーとDaily Note Pathを設定すれば、ローカルMarkdown書き出しと並行して動かせます。

---

まずはAIプロバイダーとローカルMarkdown書き出しの2つだけ設定して、数日そのまま使ってみることをおすすめします。Obsidianを使うかどうかは、`~/Downloads/Yasumaro/` に溜まったファイルを眺めながら決めても遅くありません。
