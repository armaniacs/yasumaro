# 信頼して使えるツールであるために：Yasumaro が追求する「プライバシーとセキュリティ」の設計思想

「便利さは欲しいが、自分の閲覧履歴や API キーを外部に預けるのは不安だ」

特に Obsidian のような、極めてプライベートな知識ベースを扱うユーザーにとって、データの機密性とツールへの信頼性は譲れない条件です。

Yasumaro は、最初から「ローカルファースト」と「透明性」を設計の核に据えています。v6.4 から v6.6 に至る過程で、私たちは単なる機能追加ではなく、セキュリティの基盤を根本から強化してきました。

## 「預けない」という選択：ローカルファーストなデータ管理

Yasumaro の最大の特徴は、**あなたの閲覧履歴が外部サーバーに送信されない**点にあります。

- **SQLite によるローカル保存**: 閲覧履歴データは、デバイス上の **OPFS (Origin Private File System)** 上の SQLite DB に保存されます。データはあなたの PC 内にのみ存在し、開発者ですらアクセスすることはできません。
- **3層のストレージバックエンド**: OPFS が使えない環境でも、IndexedDB VFS や `chrome.storage.local` への自動フォールバックを備えており、どのような環境でもデータが失われない堅牢な設計になっています（詳細は [ストレージアーキテクチャ guide](STORAGE_MODES.md) を参照）。
- **Obsidian への直接書き込み**: AI 要約後のデータは、Local REST API を介してあなたのローカルの Obsidian へ直接書き込まれます。クラウドを介さないため、通信経路での漏洩リスクを最小限に抑えています。

## プライバシーを制御する「プライバシーパイプライン」

AI にデータを送る際、「すべてを送信したくない」と感じるのは当然です。そこで導入したのが **プライバシーパイプライン** です。

AI に送信される前に、住所、電話番号、クレジットカード番号、メールアドレス、さらには EU 圏の税 ID など、個人情報 (PII) を自動的に検出し、`[MASKED]` に置換します。
- **確認・編集プレビュー**: AI に送信される直前に、どの箇所がマスクされたかをモーダル UI で確認でき、必要に応じて手動で修正してから送信できます。
- **プライベートページ自動検出**: HTTP ヘッダー (`Cache-Control: private` や `Set-Cookie` など) を分析し、銀行やメールなどのプライベートページを自動的に検出。誤って記録されないよう、保存前に確認ダイアログを表示します。

## 徹底したセキュリティ監査とハードニング

私たちは「自称」の安全ではなく、客観的な検証を重視しています。最近では、専門的なセキュリティ監査ツール **VulnHunter** を導入し、コードベースを徹底的にスキャンしました。

この監査を通じて、以下の強化を実施しました：
- **マスターパスワードによる暗号化**: 設定のエクスポート/インポート時に、AES-GCM (PBKDF2 100,000回反復) による強力な暗号化を実装。マスターパスワードを設定することで、API キーなどの機密情報を完全に保護します。
- **SSRF 対策の徹底**: 外部 URL をフェッチする際、内部ネットワークへの不正アクセスを遮断する厳格なバリデーションを実装しました。
- **攻撃サーフェスの削減**: 不要なエクスポートやデッドコードを徹底的に排除し、攻撃者が利用可能な隙を最小限に抑えました。

## 透明性の確保

セキュリティは「信じろ」ではなく「検証せよ」であるべきです。

Yasumaro は、どのようなデータがどこに保存され、どう処理されるかをまとめた [プライバシーポリシー](PRIVACY.md) や [PII 機能ガイド](PII_FEATURE_GUIDE.md) を公開しています。また、AI プロバイダーへの送信履歴を含む監査ログを TSV 形式でエクスポートでき、ユーザー自身が「いつ、何が送られたか」を完全に追跡可能です。

あなたの思考の軌跡を、誰にも邪魔されず、安全に蓄積する。それが Yasumaro の約束です。
