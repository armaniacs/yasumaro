# AI の「ノイズ」を削ぎ落とし、純粋なエッセンスだけを抽出する：Yasumaro の AI 戦略

AI による要約は便利ですが、多くの人が直面するのが「AI っぽさ」や「不要な情報の混入」という課題です。

「要約はしてくれたけれど、冒頭の『承知いたしました。以下に要約します』という挨拶や、末尾の『この記事についてさらに詳しく知りたい場合は...』という定型文が邪魔だ」
「Web サイトの linc や linc のタイトルが要約に混じってしまい、肝心の内容が薄まっている」

Yasumaro は、単に AI API を叩くのではなく、**「AI が最高のパフォーマンスを発揮するための前処理と後処理」** に徹底的にこだわりました。

## 1. ノイズを徹底排除する「AI 要約クレンジング」

v6.6 の目玉機能の一つが、高度な **AI 要約クレンジング (AI Summary Cleansing)** です。

Web サイトの構造は千差万別ですが、多くのサイトには「共通のノイズ」が存在します。
- **汎用的なノイズ**: 広告、ナビゲーション、ソーシャルウィジェット、JSON-LD、メタデータ、遅延読み込み要素などを削除。
- **日本語サイト特化オプション**: SWELL や Cocoon 等の国産テーマ固有のクラス、アフィリエイト広告、吹き出しのキャラクター名、ニュースメディアの記者クレジットなどをピンポイントで除去。
- **プラットフォーム固有のノイズ**: YouTube のコメント弾幕や、5ch/be の ID などのノイズを排除。

これらの処理を AI に渡す前に実行することで、AI は「純粋な本文」だけを読み込み、より精緻でノイズのない要約を生成します。詳細な処理順序は [クレンジングの順番](CLEANSING_ORDER.md) で確認できます。

## 2. 「狙い撃ち」で抽出するホワイトリストモード

一部のサイト（Togetter, 5chまとめ, Yahoo!知恵袋, なろう, クックパッド等）では、周辺ノイズが激しすぎて、通常の「削除方式」では本文を綺麗に残せません。

そこで導入したのが **ドメイン別ホワイトリスト抽出モード** です。
サイトごとに「ここが本文である」という特定のクラスや ID（例: なろうの `#novel_honbun`）を定義し、そこだけを狙い撃ちして抽出します。これにより、どんなにノイズが多いサイトからでも、純粋な本文だけを完璧に抜き出すことが可能です。

## 3. 自由自在な AI プロバイダー戦略

特定の AI モデルに縛られる時代は終わりました。Yasumaro は、柔軟なプロバイダー切り替えをサポートしています。

- **優先度フォールバック (1〜3位)**: メインの AI がエラーを返したり、要約が短すぎたり（`Summary Min Length` 未満）した場合、自動的に次順位のプロバイダーへ切り替えて再試行します。これにより、高い可用性と品質を同時に確保しています。
- **多様な接続先**: Google Gemini, OpenAI, Anthropic (via Compatible API) はもちろん、Ollama や LM Studio といったローカル LLM も利用可能。
- **コストと効率の可視化**: ダッシュボードの診断パネルで、月間の API 呼び出し回数と消費トークンを可視化。コストを意識した運用が可能です。

## 結論：AI は「道具」であり、制御してこそ価値がある

AI の出力が不完全なのは、AI の性能不足ではなく、入力データの汚れや制御の不足であることがほとんどです。

Yasumaro は、徹底したクレンジングと柔軟なプロバイダー制御により、AI を「魔法の箱」ではなく「制御可能な高精度な抽出エンジン」へと変えました。あなたの知的な探求を加速させる、最高の AI パートナーとしてぜひ活用してください。
