## Migration Steps

### Phase 1: Create `vitest.setup.ts`

Create `vitest.setup.ts` based on `jest.setup.ts`:

| From | To |
|------|----|
| `jest.fn()` | `vi.fn()` |
| `jest.clearAllMocks()` | `vi.clearAllMocks()` |
| `jest.Mock<>` | `Mock<>` from vitest |
| `global.alert = jest.fn()` | `global.alert = vi.fn()` |

Preserve: Chrome API mocks, Web Crypto polyfill, TextEncoder/TextDecoder polyfills

→ Verify: `npm test` passes (with setup file)

### Phase 2: Convert Jest APIs in Test Files

**Approach:** Iterate - convert files, discover issues, update plan.

**Step 2.1: Pilot with 2-3 files**
- Pick simplest files first (no `jest.mock()`, no timers)
- Run each with `npx vitest run <file>`
- Document actual issues encountered
- Update this plan with findings

**Step 2.2: Refine approach based on pilot**
- Adjust conversion order if needed
- Identify files needing special handling
- Update reference tables if Vitest APIs differ

**Step 2.3: Continue with remaining files**
- Repeat for each file: convert → test → fix → verify
- Update plan if new patterns discovered

→ Verify after each file: `npx vitest run <file>` passes

### Phase 3: Update `src/__tests__/types.ts`

| From | To |
|------|----|
| `import type { Mock } from 'jest-mock'` | `import type { Mock } from 'vitest'` |
| `jest.Mock<>` | `Mock<>` or `ReturnType<typeof vi.fn>` |
| `jest.MockedFunction<>` | `MockedFunction<>` from vitest |
| `jest.MockedClass<>` | `MockedClass<>` from vitest |

→ Verify: `npm run type-check` passes

### Phase 4: Update `vitest.config.ts` for Vitest 4.x

Fix deprecation warning: `poolOptions` removed in Vitest 4.x

```typescript
// Before (Vitest 1.x style)
pool: 'threads',
poolOptions: { threads: { maxThreads: 4, minThreads: 1 } }

// After (Vitest 4.x style)
pool: 'threads',
threads: { maxThreads: 4, minThreads: 1 }
```

→ Verify: Warning removed on `npm test`

### Phase 5: Remove Jest dependencies

```bash
npm remove @jest/globals @types/jest jest jest-environment-jsdom ts-jest
```

→ Verify: `npm test` passes

### Phase 6: Remove Jest config files

```bash
rm jest.config.cjs jest.setup.ts
```

→ Verify: `npm run validate` passes

### Phase 7: Update remaining scripts

Update `test:false-positive-rate` and `test:gate:false-positive` to use Vitest or remove if obsolete.

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Jest API conversion breaks tests | Manual approach - verify each file individually |
| Vitest 4.x incompatibility | Update config after confirming basic functionality |
| types.ts update breaks imports | Update after confirming file conversion works |
| Chrome mocks unavailable | Ensure vitest.setup.ts provides all required mocks |
| Timer API differences | Use `vi.useFakeTimers()` and related APIs explicitly |
