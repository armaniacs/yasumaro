## Key Fixes Applied

### 1. Arrow Function Constructor Issue (Critical)

**Problem:** Vitest's `vi.fn().mockImplementation(() => ({...}))` returns an arrow function, but `new` requires a constructor function.

**Solution:** Use `function() { this.property = value }` pattern:

```typescript
// Before (Jest) - worked because Jest didn't care about arrow functions
privacy.PrivacyPipeline.mockImplementation(() => ({
  process: vi.fn().mockResolvedValue({ summary: 'Test' })
}));

// After (Vitest) - must use function() for constructor compatibility
privacy.PrivacyPipeline.mockImplementation(function(this: any) {
  this.process = vi.fn().mockResolvedValue({ summary: 'Test' });
});
```

**Affected files:** recordingLogic.test.ts, recordingLogic-whitelist-bypass.test.ts, integration-recording.test.ts, checkTrustDomainStep.test.ts, etc.

### 2. jest.clearModuleRegistry() → vi.resetModules()

```typescript
// Before
jest.clearModuleRegistry?.() || vi.resetModules();

// After
vi.resetModules();
```

### 3. jest.setTimeout() → Removed

Jest's `jest.setTimeout()` is unnecessary in Vitest.

### 4. vi.mock() with Variables (Hoisting Issue)

**Problem:** `vi.mock` is hoisted, so variables defined in test body aren't available in factory.

**Solution:** Use `vi.hoisted()`:

```typescript
// Before (ReferenceError)
const mockInitialize = vi.fn();
vi.mock('../trustDb/trustDb.js', () => ({
  getTrustDb: () => ({ initialize: mockInitialize })
}));

// After (works)
const { mockInitialize } = vi.hoisted(() => ({
  mockInitialize: vi.fn()
}));
vi.mock('../trustDb/trustDb.js', () => ({
  getTrustDb: () => ({ initialize: mockInitialize })
}));
```

### 5. require() → vi.mocked()

Mixed `require()` and `import` in tests caused issues. Standardized on `vi.mocked(module).functionName`.
